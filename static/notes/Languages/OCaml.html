WTF is OCaml?

OCaml (Objective Caml) is a functional programming language in the ML family. It’s known for:

  • Strong static type system with type inference
  • Functional programming (immutability, higher-order functions)
  • Fast compilation and performance
  • Used extensively for Compilers and formal verification

Think of it as the language computer science researchers use when they want to prove their code is correct.

Key Features

1. Type Inference

The Compiler figures out types - no need to write them (unlike C++).

2. Algebraic Data Types (ADTs)

3. Immutability by Default

4. Module System

Modules provide namespacing and abstraction.

5. Functors (Parameterized Modules)

Like C++ templates but for modules.

Why OCaml is Popular in Certain Niches

1. Compiler Development

Famous compilers written in OCaml:

  • Rust compiler (originally)
  • Compiler for Rust’s borrow checker analysis
  • Flow (JavaScript type checker)
  • Hack (PHP dialect)
  • Coq (proof assistant)

Why? Strong types, pattern matching, fast compilation.

2. Formal Verification

Used with proof assistants like Coq.

3. Financial Systems

  • Jane Street (major trading firm) uses OCaml for everything
  • Critical systems where correctness matters
  • Type safety prevents costly bugs

4. WebAssembly Tools

  • ReScript (formerly BuckleScript) - Compiles OCaml to JavaScript
  • wasm_of_ocaml - Compiles OCaml to WebAssembly

Real-World OCaml Projects

1. Jane Street’s Trading Systems

Type safety prevents mixing up prices, quantities, symbols.

2. Infer (Facebook’s Static Analyzer)

Analyzes code for bugs (null pointer, memory leaks, etc.).

Written in OCaml for strong type guarantees.

3. Coq (Proof Assistant)

Coq is written in OCaml and generates OCaml code.

4. MirageOS (Unikernel OS)

Entire Operating-System written in OCaml.

Minimal attack surface - type safety prevents many vulnerabilities.

Learning Curve

Easy → Hard:

  ↑                             ↑

  • Different paradigm (functional)
  • Strong type system
  • Pattern matching everywhere
  • Immutability by default

But: Once you “get it,” you’ll never want to go back to null pointer errors.

The Big Takeaway

OCaml in a Nutshell

OCaml is a functional programming language known for strong type safety and use in Compiler development.

Key features:

  • Strong type inference (like Rust but automatic)
  • Algebraic Data Types (enums with data)
  • Pattern matching
  • Immutability by default
  • Powerful module system

Used for:

  • Compilers (Rust, Flow, Hack)
  • Formal verification (Coq)
  • Financial systems (Jane Street)
  • Static analysis tools (Infer)

Why use it:

  • Type safety prevents bugs
  • Fast compilation
  • Great for compiler/language work
  • Formal verification possible

Influenced: Rust’s type system, enums, pattern matching

The rule: OCaml is the language academics and compiler developers use when they need strong correctness guarantees. If you’re building a Compiler, type checker, or proof assistant, OCaml is a natural choice. Rust borrowed many of its best ideas.
