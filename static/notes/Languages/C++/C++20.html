WTF is C++20?

C++20 is the biggest update to C++ since C++11. It’s not just a few tweaks - it’s a fundamental shift in how you write C++ code.

Think: going from manual transmission to automatic. The car still works the same way, but driving it feels completely different.

The Evolution Analogy

The Language Evolution

C++98/03: The Original

Manual memory management, templates without constraints, header files everywhere. Like programming with stone tools - it works, but it’s painful.

C++11/14: The Renaissance

Smart pointers, lambdas, move semantics, auto keyword. Like discovering metal tools - way better!

C++17: The Refinement

std::optional, std::variant, structured bindings. Polishing the metal tools to a shine.

C++20: The Revolution

Concepts, coroutines, ranges, modules. Like jumping from hand tools straight to power tools. Game changer.

That’s the leap C++20 represents.

The Big 4 Features

1. Concepts - Template Constraints That Make Sense

Before C++20:

With C++20:

Concepts let you say “this template only works with types that have X, Y, Z”. Error messages become readable again.

2. Coroutines - Async/Await for C++

Before C++20 (callback hell):

With C++20:

co_await lets you write async code that looks like normal sequential code. No more Callback-Hell.

3. Ranges - Composable Algorithms

Before C++20:

With C++20:

Ranges let you chain operations like Rust iterators or JavaScript array methods. Clean and expressive.

4. Modules - Goodbye Header Files

Before C++20:

Every #include literally copy-pastes text. Compile times explode.

With C++20:

Modules are pre-compiled. Include a module 1000 times? Compiled once. Massive build time improvements.

Why C++20 Matters

1. Better Than C++17

  • Concepts fix template error messages (finally!)
  • Coroutines enable modern async code
  • Ranges make algorithms composable
  • Modules fix compile times

2. Competitive With Modern Languages

  • Coroutines match Rust’s async/await
  • Ranges feel like Rust iterators
  • Concepts provide Rust-like trait bounds
  • Still has C++ performance and control

3. The Foundation for C++23/26

C++20 is the base for future improvements:

  • C++23 adds std::expected (like Rust’s Result)
  • C++26 will build on coroutines and ranges
  • Modules will become the standard (no more headers!)

C++20 is hard to learn because it changes so much. But once you get it, you won’t want to go back.

Compiler Support

As of 2024:

  • GCC 10+: Good support
  • Clang 13+: Good support
  • MSVC 2019+: Good support

Coroutines and modules took time to stabilize, but they’re production-ready now.

Migration Strategy

You don’t have to rewrite everything. Adopt incrementally:

  1. Start with ranges - Easy win, clear code
  2. Add concepts to templates - Better errors
  3. Try std::format - Safer string formatting
  4. Experiment with coroutines - For async code
  5. Modules last - Requires build system changes

The Big Takeaway

C++20 in a Nutshell

C++20 is the biggest C++ update in a decade. Four major features:

  1. Concepts - Template constraints that actually make sense
  2. Coroutines - async/await for C++ (co_await)
  3. Ranges - Composable, chainable algorithms
  4. Modules - Replace header files, fix compile times

Plus: spaceship operator, std::format, std::span, consteval, and more.

It’s a steep learning curve, but C++20 makes C++ feel like a modern language again.

The rule: If you’re starting a new C++ project in 2024+, use C++20 as your baseline. The old way of writing C++ is officially obsolete.
