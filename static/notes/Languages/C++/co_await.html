The Three Coroutine Keywords

C++20 coroutines have three special keywords. Using any one makes a function a coroutine:

1. co_await - Suspend and Wait

What happens:

  1. Call http_get(url) - starts async operation
  2. co_await suspends the function (saves state)
  3. Control returns to caller (can do other work)
  4. When data arrives, resumes at the same line
  5. Continues with response.body()

2. co_yield - Suspend and Return Value

What happens:

  • co_yield a returns value to caller
  • Function suspends (keeps state: a, b)
  • Caller asks for next value
  • Function resumes from where it left off

3. co_return - Return Final Value

Like regular return, but for coroutines.

The Big Takeaway

co_await in a Nutshell

co_await is C++20‚Äôs operator for suspending a coroutine until an async operation completes.

Three keywords make a function a coroutine:

  • co_await - Suspend and wait for result
  • co_yield - Suspend and return value (generators)
  • co_return - Return final value

Benefits:

  • Escape Callback-Hell
  • Write async code that looks synchronous
  • Clean error handling with try/catch
  • Lightweight (cheaper than threads)

The Compiler transforms your code into a state machine that can suspend and resume.

The rule: If you‚Äôre doing async I/O in C++20+, use co_await. It‚Äôs the modern way to handle asynchronous operations without drowning in callbacks.
