Virtual Keycode vs Unicode: Key vs Character

Virtual Keycode: Identifies which physical key was pressed Unicode: Identifies which character should appear

Visual Example

Imagine pressing the key next to ‘L’ on a QWERTY keyboard:

┌─────────────────────────────────────────┐
│  Physical Keyboard Action               │
│  User presses the ';' key (US layout)   │
└─────────────────────────────────────────┘
              ↓
    ┌─────────────────────────┐
    │   Virtual Keycode       │
    │   VK_OEM_1 (0xBA)       │
    │   "The key at this      │
    │    position"            │
    └─────────────────────────┘
              ↓
    ┌─────────────────────────┐
    │   OS Keyboard Layout    │
    │   Translates based on   │
    │   active layout         │
    └─────────────────────────┘
              ↓
    ┌─────────────────────────┐
    │   Unicode Character     │
    │   US: ';' (U+003B)      │
    │   French: 'm' (U+006D)  │
    │   German: 'ö' (U+00F6)  │
    └─────────────────────────┘

Why Windows Has Both Methods

SendInput with Virtual Keycodes:

  • Application sees: “The ‘A’ key was pressed”
  • OS translates based on layout
  • Triggers keyboard shortcuts

SendInput with KEYEVENTF_UNICODE:

  • Application sees: “Character ‘é’ was typed”
  • No layout translation needed
  • Does NOT trigger shortcuts

For Your libkynput Implementation

Your system currently uses scancodes (similar to virtual keycodes):

Scancodes are even lower level than virtual keycodes:

  • Scancode: Hardware code from keyboard controller
  • Virtual Keycode: OS abstraction of scancode
  • Unicode: Character produced after OS processing
Hardware → Scancode → Virtual Keycode → Unicode Character
Keyboard    (0x1E)      (VK_A/0x41)      ('a'/U+0061)

When to Use Each

Use Virtual Keycodes/Scancodes when:

  • Building remote desktop (simulate real keyboard)
  • Need hotkeys to work (Ctrl+C, Alt+F4)
  • Gaming (WASD movement)
  • Need exact key positions

Use Unicode when:

  • Input Method Editors (IME)
  • Text dictation/voice input
  • Clipboard paste as typing
  • Don’t care about shortcuts

Key Takeaway

Virtual keycodes answer: “Which key did the user press?” Unicode answers: “Which character should appear on screen?”

For remote desktop/game streaming like libkynput, you want scancodes/keycodes because you’re transmitting keyboard hardware events, not just text. This preserves shortcuts, game controls, and layout independence.
