Capturing Virtual Keyboard Input for Remote Desktop on Android

The Problem

You’re building a remote desktop client (RDP, VNC, or similar) on Android. When users type on their virtual keyboard, you need to:

  1. Capture the Unicode characters they type
  2. Convert those characters to scan codes or key events
  3. Send those events to the remote Windows/Linux machine

The remote machine expects scan codes (like a physical keyboard), but Android’s virtual keyboard only gives you Unicode characters.

The Bottom Line

For remote desktop on Android:

  1. Capture virtual keyboard input using InputConnection
  2. Store and log everything as UTF-8
  3. Translate Unicode characters to scan codes for ASCII
  4. Send as Unicode for non-ASCII (emoji, CJK, etc.)
  5. Always use UTF-8 for clipboard, configs, and logs

The virtual keyboard gives you Unicode characters. Your job is to translate those to what the remote machine expects, whether that’s scan codes or Unicode input events.

UTF-8 everywhere still applies: every string you log, save, or transmit should be explicitly UTF-8 encoded.
