Why UTF-32 on the Wire?

The Wire Protocol Uses UTF-32 Because:

  1. Fixed Size: Always exactly 4 bytes per character

    • No variable-length complexity
    • Easy to parse: just read 4 bytes

  2. Rust Native: Rustâ€™s char type is UTF-32

  3. No Surrogate Pair Handling: The wire protocol never deals with surrogates

    • Surrogates are a UTF-16 implementation detail
    • UTF-32 has no surrogates (each value is a complete codepoint)
