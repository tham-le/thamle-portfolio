libkynput Project - Development Notes

This folder contains technical notes and implementation guides for the libkynput project - a Rust library for cross-platform keyboard and mouse input capture and injection, used in remote desktop and game streaming applications.

What is libkynput?

libkynput is a systems programming library that:

  • Captures keyboard/mouse input on client devices (iPad, Android, etc.)
  • Serializes input events into network packets
  • Sends them over the network to a host computer
  • Injects the input into the host OS (Windows, Linux, macOS)

Think Parsec, Steam Remote Play, or Moonlight architecture.

Architecture Overview

┌──────────────┐         Network          ┌──────────────┐
│    Client    │  ───────────────────→   │     Host     │
│  (Captures)  │   InputPacket bytes     │  (Injects)   │
│              │                          │              │
│  iOS/Android │  ←───────────────────   │ Windows/Linux│
└──────────────┘    Video frames         └──────────────┘

Input Flow:

  1. User types on virtual keyboard → OS gives char
  2. Client wraps in InputPacket → Serializes to bytes
  3. Sends over network (TCP/UDP/WebRTC)
  4. Host deserializes → Injects via platform API
  5. Character appears in focused application

Technologies Used

  • Rust - Systems programming language
  • Windows API - SendInput(), registry access
  • FFI - Interfacing with C libraries
  • Network protocols - Custom serialization
  • Platform-specific APIs - Windows, Linux (X11/Wayland), macOS
