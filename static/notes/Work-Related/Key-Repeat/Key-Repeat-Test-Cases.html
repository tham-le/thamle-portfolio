Complete Test Cases for Key Repeat Implementation

Basic Functionality Tests

1. Single Key Repeat

Test: Hold ‘a’ key

  • Expected:
    • First ‘a’ appears immediately
    • After 250-1000ms delay (based on settings): ‘a’ starts repeating
    • Repeats every 33-400ms (based on settings)
    • Result: aaaaaaaaaa
  • Verifies: Basic repeat mechanism works

2. Quick Tap (No Repeat)

Test: Press and immediately release ‘a’

  • Expected: Only one ‘a’ appears, no repeats
  • Verifies: Initial delay prevents unwanted repeats

3. Multiple Different Keys Sequentially

Test: Hold ‘a’, release, hold ‘b’, release, hold ‘c’

  • Expected:
    • aaaaaaa → stop → bbbbbbb → stop → ccccccc
  • Verifies: Each key repeats independently when held alone

Key Switching Tests

4. Switch Keys While Holding

Test: Hold ‘a’, then press ‘s’ (while still holding ‘a’)

  • Expected:
    • aaaaaaa → press ‘s’ → s (stops ‘a’ repeat)
    • Now only ‘s’ repeats: sssssss
    • NOT: asasasasas
  • Verifies: Only most recent key repeats

5. Switch Keys Multiple Times

Test: Hold ‘a’, press ‘d’, press ‘f’

  • Expected:
    • aaaaaaa → d → ddddd → f → fffff
  • Verifies: Each new key becomes the repeating key

6. Release Second Key, First Still Held

Test: Hold ‘a’, press ‘s’, release ‘s’ (still holding ‘a’)

  • Expected:
    • aaaaaaa → s → sssss → release ‘s’ → stops repeating
    • Does NOT go back to repeating ‘a’
  • Verifies: Repeating stops when repeating key is released (matches standard keyboard behavior)

Modifier Key Tests

7. Shift + Letter

Test: Hold ‘a’, press Shift (while holding ‘a’)

  • Expected:
    • aaaaaaa → press Shift → AAAAA (uppercase, continues repeating)
  • Verifies: Shift modifies the repeating character but doesn’t become the repeating key

8. Shift Then Letter

Test: Hold Shift, press ‘a’

  • Expected:
    • Press Shift (nothing happens) → hold ‘a’ → AAAAA
  • Verifies: Shift alone doesn’t repeat, ‘a’ repeats as uppercase

9. Ctrl + Letter

Test: Hold ‘a’, press Ctrl

  • Expected:
    • aaaaaaa → press Ctrl → Ctrl+A events repeat (might select all repeatedly in app)
  • Verifies: Ctrl modifies repeating key, doesn’t become repeating key

10. Alt + Letter

Test: Hold Alt, press ‘a’

  • Expected:
    • Alt alone doesn’t repeat → press ‘a’ → Alt+A repeats
  • Verifies: Alt works like Shift and Ctrl

11. Win + Letter

Test: Hold ‘a’, press Win key

  • Expected:
    • aaaaaaa → press Win → Windows shortcuts might trigger
  • Verifies: Win key is treated as modifier

12. Hold Only Shift

Test: Hold Shift for 2 seconds

  • Expected: Nothing happens, no repeat Shift events
  • Verifies: Modifiers never repeat

13. Hold Only Ctrl

Test: Hold Ctrl for 2 seconds

  • Expected: Nothing happens
  • Verifies: Ctrl doesn’t repeat

Windows Shortcut Tests

14. Ctrl+Alt+Del

Test: Hold Ctrl, hold Alt, press Del

  • Expected:
    • Security screen appears (handled by handle_windows_shortcut)
    • No repeating
  • Verifies: Special shortcuts still work

15. Win+L

Test: Hold Win, press L

  • Expected:
    • Workstation locks
    • No repeating
  • Verifies: Lock shortcut works

Timing Tests

16. Respect Initial Delay

Test: Hold ‘a’, measure time until first repeat

  • Expected:
    • Based on registry setting (250-1000ms)
    • Default usually 500ms
  • Verifies: Initial delay from Windows settings is respected

17. Respect Repeat Interval

Test: Hold ‘a’, measure time between repeats

  • Expected:
    • Based on registry setting (33-400ms)
    • Default usually 33ms for fast setting
  • Verifies: Repeat speed from Windows settings is respected

18. Immediate Stop on Release

Test: Hold ‘a’ until repeating, then release quickly

  • Expected:
    • Last character appears
    • At most 1 extra character after release (due to network latency)
    • NOT: Several characters after release
  • Verifies: Quick reaction to KeyUp

Client Behavior Tests

19. Client Sends Repeats (Most Clients)

Test: Client that sends: KeyDown → KeyDown → KeyDown → KeyUp

  • Expected:
    • Server filters client repeats
    • Generates own repeats based on server timing
    • User sees consistent timing
  • Verifies: Client repeats are ignored

20. Client Doesn’t Send Repeats (Some Clients)

Test: Client that sends: KeyDown → (nothing) → KeyUp

  • Expected:
    • Server generates repeats
    • User sees repeats at correct timing
  • Verifies: Server emulates when client doesn’t send repeats

Edge Cases

21. Very Fast Typing

Test: Type ‘asdf’ very quickly (each key < 100ms)

  • Expected:
    • asdf appears
    • No repeats (released before delay expires)
  • Verifies: Fast typing doesn’t trigger repeats

22. Hold Many Keys Simultaneously

Test: Hold ‘a’, ‘s’, ‘d’, ‘f’ at the same time

  • Expected:
    • Only the most recent key repeats
    • If pressed in order: ‘f’ repeats
  • Verifies: Only one key repeats even with many held

23. Spam KeyDown (Client Bug Simulation)

Test: Client sends KeyDown 100 times per second for ‘a’

  • Expected:
    • Only first KeyDown processed
    • Rest ignored
    • Server generates own repeats at correct rate
  • Verifies: Protection against buggy clients

24. Extended Keys (Arrow Keys)

Test: Hold down arrow key (scancode >= 0xE000)

  • Expected:
    • Arrow repeats normally
    • KEYEVENTF_EXTENDEDKEY flag set
  • Verifies: Extended keys work correctly

25. Numpad Keys

Test: Hold Numpad 5

  • Expected:
    • ‘5’ repeats (if NumLock on)
    • Or does nothing (if NumLock off)
  • Verifies: Numpad keys work

Performance Tests

26. CPU Usage When Idle

Test: No keys pressed, measure CPU

  • Expected:
    • Thread sleeps (no timeout wakeups)
    • ~0% CPU usage
  • Verifies: Efficient when idle

27. CPU Usage When Holding One Key

Test: Hold ‘a’ for 10 seconds, measure CPU

  • Expected:
    • Minimal CPU (~0.05-0.1%)
    • Thread wakes only when deadline reached
  • Verifies: Efficient repeat generation

28. Latency When Packets Arrive

Test: Send packets while key is repeating

  • Expected:
    • Packets processed immediately
    • No delay added by repeat mechanism
  • Verifies: Repeats don’t impact packet processing

Network/Latency Tests

29. High Network Latency

Test: Simulate 200ms network delay

  • Expected:
    • Initial key appears after 200ms
    • Repeats continue smoothly at server rate
    • KeyUp processed after 200ms delay
    • Max 1-2 extra characters after release
  • Verifies: Works correctly with network delay

30. Packet Loss

Test: Drop 50% of repeat KeyDown packets from client

  • Expected:
    • Server ignores client repeats anyway
    • Generates own repeats
    • No impact on user experience
  • Verifies: Robust against packet loss

Gaming Tests

31. WASD Movement

Test: In a game, hold ‘W’ to move forward

  • Expected:
    • Smooth continuous movement
    • No stuttering
    • Character moves at constant speed
  • Verifies: Games using GetAsyncKeyState() work

32. Space to Jump

Test: In a game, hold Space

  • Expected:
    • Jump once (if game uses KeyDown event)
    • Or continuous jump attempts (if game polls state)
    • NOT: Multiple jump events causing double-jump
  • Verifies: Action keys work correctly

33. Hold ‘W’, Press ‘S’ (Opposite Direction)

Test: In game, hold ‘W’ (forward), press ‘S’ (backward)

  • Expected:
    • Movement changes to backward
    • No stuttering
  • Verifies: Direction changes work smoothly

Text Editor Tests

34. Hold Backspace

Test: In text editor, hold Backspace

  • Expected:
    • Characters delete continuously
    • Respects Windows repeat settings
  • Verifies: Special keys like Backspace repeat

35. Hold Delete

Test: In text editor, hold Delete

  • Expected:
    • Characters delete forward continuously
  • Verifies: Delete key repeats

36. Hold Space

Test: In text editor, hold Space

  • Expected:
    • Spaces appear continuously
  • Verifies: Space bar repeats

37. Hold Arrow Key

Test: In text editor, hold Right arrow

  • Expected:
    • Cursor moves continuously right
  • Verifies: Navigation keys repeat

Registry Settings Tests

38. Change Windows Keyboard Delay

Test:

  1. Set Windows keyboard delay to 0 (short)
  2. Restart server
  3. Hold ‘a’
  • Expected: Repeat starts after ~250ms
  • Test Again: Set delay to 3 (long), restart, test
  • Expected: Repeat starts after ~1000ms
  • Verifies: Reads registry settings correctly

39. Change Windows Keyboard Speed

Test:

  1. Set Windows keyboard speed to 0 (slow)
  2. Restart server
  3. Hold ‘a’
  • Expected: Repeat interval ~400ms
  • Test Again: Set speed to 31 (fast), restart, test
  • Expected: Repeat interval ~33ms
  • Verifies: Respects speed settings

Stress Tests

40. Hold Key for 60 Seconds

Test: Hold ‘a’ for one minute

  • Expected:
    • Consistent repeat rate throughout
    • No memory leaks
    • No CPU spike
  • Verifies: Stability over time

41. Rapidly Press/Release 100 Times

Test: Press ‘a’ 100 times rapidly (tap-tap-tap)

  • Expected:
    • 100 ‘a’ characters appear
    • No repeats (each press < delay time)
  • Verifies: Handles rapid input

42. Server Restart During Hold

Test: Hold ‘a’, restart server, continue holding

  • Expected:
    • Graceful shutdown
    • After restart: ‘a’ starts fresh (no memory of previous hold)
  • Verifies: Clean startup/shutdown
