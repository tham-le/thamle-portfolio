Why This is the Most Efficient Solution

Performance Characteristics:

1. When messages are flowing (active use):

Message arrives → thread wakes immediately → processes → sleeps again
No 10ms wait! Instant response.

2. When holding a key (repeating):

Timeout after 10ms → on_tick() called → checks held keys → generates repeat if needed
CPU usage: ~5 microseconds every 10ms = 0.05% CPU

3. When idle (no keys held, no messages):

Timeout after 10ms → on_tick() called → held_keys is empty → returns immediately
CPU usage: ~1 microsecond every 10ms = 0.01% CPU

4. When queue is closed (shutdown):

Detects closed queue → exits loop → thread terminates cleanly
