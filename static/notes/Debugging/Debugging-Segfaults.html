A segmentation fault (segfault) happens when your program tries to access memory it shouldn’t:

  • Dereferencing NULL pointer
  • Writing to read-only memory
  • Stack overflow
  • Accessing freed memory
  • Buffer overflow

The Big Takeaway

Most segfaults are:

  1. NULL pointer dereference (40%)
  2. Buffer overflow (30%)
  3. Use-after-free (20%)
  4. Other (10%)

Essential tools:

  • GDB for stack traces
  • Valgrind for memory errors
  • ASan for fast detection

Prevention > Debugging:

  • Initialize pointers
  • Check return values
  • Use safe functions
  • Bounds checking

Pro tip: Run with valgrind and ASan regularly during development to catch bugs early!
