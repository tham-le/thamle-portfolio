Getting Started with GDB

Nov 06, 20256 min read

  • debugging
  • tools
  • gdb

GDB (GNU Debugger) is the standard debugger for C/C++/Rust programs. It lets you:

  • Pause execution at specific lines (breakpoints)
  • Step through code line-by-line
  • Inspect variables and memory
  • Analyze crashes (core dumps)

Practice Exercise

Try debugging this buggy program:

Debugging steps:

  1. Compile: gcc -g -o bug bug.c
  2. Run in GDB: gdb ./bug
  3. Set breakpoint: b process
  4. Run: r
  5. Watch the bug: display i then repeatedly n
  6. Find when i > size

The Big Takeaway

GDB is essential for:

  • Finding segfaults
  • Debugging logic errors
  • Understanding program flow
  • Analyzing crashes

Master these commands: break, run, next, print, backtrace

Pro tip: Use -g -O0 for best debugging experience

Practice regularly! GDB proficiency comes from repetition.
