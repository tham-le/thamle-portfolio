Key Implementation Details

Scancode mapping is critical: The Android app must convert Android keycodes to Scancode Set 1 (the format KeyboardKey expects). Example:

Virtual keyboard differences: Virtual keyboards send complete key events (press + release), unlike physical keyboards where you handle the hardware scan. Make sure you send both press and release events:

Packet Flow

Android Virtual Keyboard
    ↓
Android Input Capture (KeyEvent)
    ↓
Convert to Scancode Set 1
    ↓
Create KeyboardKey(scancode, pressed)
    ↓
Wrap in InputPacket(target=Host)
    ↓
Serialize to bytes (3 bytes: u16 scancode + u8 pressed)
    ↓
Send over network (TCP/Kymux)
    ↓
Host receives bytes
    ↓
Deserialize to KeyboardKey via Payload::deserialize()
    ↓
Host InputHandler injects key into OS

Common Pitfall

Missing key releases: Virtual keyboards often don’t give you explicit release events. You must synthesize them, or keys will “stick” on the host:
