Chapter 3: Machine-Level Representation of Programs

Key concept: Understanding assembly helps you write faster code and debug issues.

x86-64 Assembly Basics

Registers:

  • %rax: Return value, accumulator
  • %rbx, %rcx, %rdx: General purpose
  • %rsp: Stack pointer
  • %rbp: Base pointer
  • %rip: Instruction pointer

See also: Assembly

Function Call Convention

Caller:
1. Save caller-saved registers
2. Push arguments (right to left)
3. call function

Callee:
1. Push %rbp (save old base pointer)
2. mov %rsp, %rbp (set up new frame)
3. Allocate local variables
4. ...function body...
5. mov %rbp, %rsp (restore stack)
6. pop %rbp
7. ret

Caller-saved: %rax, %rcx, %rdx, %rsi, %rdi, %r8-r11 Callee-saved: %rbx, %r12-r15, %rbp

Chapter 4: Processor Architecture

Key concept: Modern CPUs use pipelining and parallel execution.

Pipeline Stages

1. Fetch     - Get instruction from memory
2. Decode    - Determine what to do
3. Execute   - Perform ALU operation
4. Memory    - Read/write memory
5. Write-back - Store result

Hazards:

  • Data hazard: Instruction depends on previous result
  • Control hazard: Branch prediction miss
  • Structural hazard: Resource conflict

Branch Prediction

Performance tip: Write predictable code when possible.

Chapter 12: Concurrent Programming

Key concept: Three approaches to concurrency.

Approaches

  1. Processes - Fork, heavy, isolated
  2. Threads - Shared memory, lighter, race conditions
  3. I/O Multiplexing - epoll, lightest, no parallelism

See also: std-thread, Data-Race, epoll, C10k-Problem

Key Takeaways

  1. Understanding the machine makes you a better programmer
  2. Memory hierarchy is crucial - write cache-friendly code
  3. Concurrency is hard - use proper synchronization
  4. I/O is often the bottleneck - use non-blocking I/O when appropriate
  5. Tools matter - Learn GDB, Valgrind, perf
