WTF is a Transistor?

A transistor is an electronic switch. That’s it. That’s the whole computer.

Everything else - your CPU, RAM, GPU, phone, the internet - is just billions of these switches turning on and off really fast.

Let’s understand how a transistor works, then build a computer from scratch.

The simplest explanation

Imagine a light switch:

     [Switch]
        │
[Battery]───[Light Bulb]
  • Switch CLOSED → Light ON (1)
  • Switch OPEN → Light OFF (0)

A transistor is the same thing, but:

  • ✅ No moving parts (solid state)
  • ✅ Switches in nanoseconds (billionths of a second)
  • ✅ Controlled by electricity (not your finger)
  • ✅ Microscopic (14nm = 140 atoms wide!)

This is the foundation of all computing.

Why switches = computation

Binary representation

Two states = one bit:

Switch OFF = 0
Switch ON  = 1

Multiple switches = numbers:

4 switches in a row:

[OFF][OFF][OFF][OFF] = 0000 = 0
[OFF][OFF][OFF][ON]  = 0001 = 1
[OFF][OFF][ON][OFF]  = 0010 = 2
[OFF][OFF][ON][ON]   = 0011 = 3
[OFF][ON][OFF][OFF]  = 0100 = 4
...
[ON][ON][ON][ON]     = 1111 = 15

8 switches = 1 byte (0-255) 64 switches = 64-bit number (0 to 18,446,744,073,709,551,615)

See: Binary

Logic from switches

You can build logic gates by combining switches:

AND gate (both switches must be ON):

Switch A ──┬──── Output
Switch B ──┘

A=0, B=0 → Output=0
A=0, B=1 → Output=0
A=1, B=0 → Output=0
A=1, B=1 → Output=1

OR gate (either switch ON):

Switch A ──┬
           ├──── Output
Switch B ──┘

A=0, B=0 → Output=0
A=0, B=1 → Output=1
A=1, B=0 → Output=1
A=1, B=1 → Output=1

From these, you can build EVERYTHING:

  • Addition, subtraction, multiplication
  • Memory (flip-flops)
  • CPU (ALU + registers + control)
  • Graphics, networking, AI

All from switches!

See: Logic-Gate

How a transistor works

The mechanical switch analogy

Old-school relay (1940s):

Control wire ──→ [Electromagnet]
                      │
                   [Metal arm] ← Pulled down when current flows
                      │
[Input]───────────[Contact]───────[Output]

When current flows through control wire:

  1. Electromagnet activates
  2. Pulls metal arm down
  3. Completes circuit from input to output

Problem: Slow (milliseconds), wears out, huge, power-hungry

Solution: Transistor (1947)

Transistor types

MOSFET (Metal-Oxide-Semiconductor Field-Effect Transistor)

Modern transistors are MOSFETs - the kind in your CPU.

       Gate (G)
          │
          ├─ [Oxide layer]
          │
Source ──[Channel]── Drain
  (S)              (D)

Three terminals:

  • Source (S): Electricity comes from here
  • Drain (D): Electricity flows to here
  • Gate (G): Controls whether current flows

How it works:

Gate voltage LOW (0V):
  Channel is non-conductive
  No current flows from Source to Drain
  Transistor = OFF

Gate voltage HIGH (5V, 3.3V, 1.8V, etc.):
  Electric field makes channel conductive
  Current flows from Source to Drain
  Transistor = ON

It’s voltage-controlled! No moving parts.

N-channel vs P-channel

NMOS (N-channel):

  • Gate HIGH → Transistor ON (conducts)
  • Gate LOW → Transistor OFF

PMOS (P-channel):

  • Gate LOW → Transistor ON (conducts)
  • Gate HIGH → Transistor OFF

Modern CPUs use CMOS (Complementary MOS) = NMOS + PMOS together.

Why? Ultra-low power consumption (only uses power when switching, not when idle).

The physics (simplified)

Semiconductors

What is silicon?

Periodic table:

     C (Carbon)
     │
     Si (Silicon) ← 4 valence electrons
     │
     Ge (Germanium)

Silicon properties:

  • Not a conductor (like copper)
  • Not an insulator (like rubber)
  • Semiconductor (in between)

Pure silicon at room temperature:

  • Very few free electrons
  • Barely conducts electricity

Doping: Creating N-type and P-type

To make silicon useful, we “dope” it (add impurities).

N-type silicon (Negative):

Add phosphorus (P) atoms
Phosphorus has 5 valence electrons (silicon has 4)
Extra electron is free to move
→ Conducts electricity

P-type silicon (Positive):

Add boron (B) atoms
Boron has 3 valence electrons
Creates "holes" (missing electrons)
→ Conducts electricity (holes move)

The PN junction

Join N-type and P-type:

N-type │ P-type
───────┼───────
       │← Junction
  ─  ─ │ +  +
  ─  ─ │ +  +

At the junction:

  • Electrons from N-type flow to P-type
  • Fills holes
  • Creates depletion zone (no charge carriers)
  • Acts as insulator

Apply voltage:

  • Forward bias: Reduces depletion zone → Current flows
  • Reverse bias: Increases depletion zone → No current

This is a diode!

Adding the Gate (MOSFET)

Put an insulated gate above the junction:

      [Gate] ← Metal
        │
      [Oxide] ← Insulator (SiO₂)
        │
[N]───[P]───[N]  ← Silicon
 S          D

Gate voltage affects the channel:

  • Gate LOW: P-type remains P-type (non-conductive between N regions)
  • Gate HIGH: Electric field converts P-type to N-type in channel
    • Now N-S-N is all N-type → Current flows!

This is a transistor!

Building a logic gate from transistors

NOT gate (Inverter)

Simplest gate - inverts input:

         Vcc (+5V)
          │
          │
       [PMOS]
          │
Input ────┼──── Output
          │
       [NMOS]
          │
         GND (0V)

Truth table:

Input=0 → PMOS ON, NMOS OFF → Output connected to Vcc → Output=1
Input=1 → PMOS OFF, NMOS ON → Output connected to GND → Output=0

Input=0, Output=1 (inverted!)

NAND gate

Two transistors in series:

         Vcc
          │
       [PMOS A]
          │
       [PMOS B]
          │
A ────────┼──── Output
          │
B ────────┼
          │
       [NMOS A]
          │
       [NMOS B]
          │
         GND

Truth table:

A=0, B=0 → Both PMOS ON → Output=1
A=0, B=1 → One PMOS ON → Output=1
A=1, B=0 → One PMOS ON → Output=1
A=1, B=1 → Both PMOS OFF, both NMOS ON → Output=0

NAND is universal! You can build ANY logic gate from NAND gates.

See: Logic-Gate

The journey from sand to CPU

1. Raw silicon (sand)

Silicon is everywhere:

  • Beach sand (SiO₂ - silicon dioxide)
  • 27% of Earth’s crust

Purification:

Sand (SiO₂)
  → Heat to 2000°C with carbon
  → Produces liquid silicon (98% pure)
  → Czochralski process (grow crystal)
  → 99.9999999% pure silicon (9 nines!)

Result: Silicon ingot (cylinder, 300mm diameter)

2. Wafer slicing

Cut ingot into wafers:

Silicon ingot
  → Diamond saw
  → 300mm diameter, 0.75mm thick wafers
  → Polish to atomic smoothness

One wafer = hundreds of CPUs

3. Photolithography (making transistors)

Like printing, but at nanometer scale:

  1. Coat wafer with photoresist (light-sensitive chemical)
  2. Shine UV light through mask (pattern of circuit)
  3. Exposed areas become soluble
  4. Wash away exposed photoresist
  5. Etch away exposed silicon
  6. Deposit materials (metals, insulators)
  7. Repeat 50+ times (different layers)

Result: Billions of transistors per chip

Feature size:

  • 1990s: 500nm (0.5 microns)
  • 2000s: 90nm
  • 2010s: 14nm
  • 2020s: 3nm (TSMC, Apple M-series)

3nm = ~15 silicon atoms wide!

4. Doping

Ion implantation:

High-energy beam of ions
  → Phosphorus (for N-type)
  → Boron (for P-type)
  → Penetrates silicon surface
  → Creates N/P regions

5. Metallization

Connect transistors with wires:

  • Multiple layers of copper
  • Separated by insulators
  • 10-15 metal layers in modern CPUs

6. Testing and packaging

Test wafer:

  • Probe each chip
  • Mark defective chips

Cut wafer into dies:

  • Each die = one CPU

Package chip:

  • Mount die in package
  • Wire bond to pins
  • Seal with heat spreader

Your CPU is born!

Modern transistor counts

Evolution:

1971: Intel 4004        2,300 transistors
1978: Intel 8086       29,000 transistors
1993: Pentium          3.1 million
2006: Core 2 Duo       291 million
2019: AMD Ryzen 9     3.9 billion
2023: Apple M2 Ultra  134 billion transistors!

134 billion switches, all switching billions of times per second.

How small are they?

3nm transistor:

  • 3 nanometers = 0.000003 millimeters
  • Width of ~15 silicon atoms
  • Quantum effects become significant!

A human hair:

  • 100,000 nanometers thick
  • 33,000 transistors fit across one hair!

Comparison:

If a transistor was the size of a marble (1cm):
  → Human hair would be 333 meters thick
  → Human would be 333 km tall

We’re approaching the limits of physics.

Why transistors matter

Moore’s Law

Observation (Gordon Moore, 1965):

“The number of transistors on a chip doubles every 18-24 months”

This held true for 50+ years!

Consequences:

  • Computers get 2× faster every 2 years
  • At same price
  • Enabled: smartphones, internet, AI

Now slowing down:

  • Approaching atomic limits
  • Quantum tunneling (electrons leak through barriers)
  • Heat dissipation problems
  • Future: 3D stacking, new materials (graphene, carbon nanotubes)

See: CPU

Power consumption

Dynamic power (switching):

P = C × V² × f

C = capacitance (how much charge to store)
V = voltage
f = frequency (how often switching)

Why modern CPUs use low voltage:

  • 5V → 1.8V → 0.9V
  • Power ∝ V² → Reducing voltage saves HUGE power

Static power (leakage):

  • Transistors leak current even when “off”
  • Gets worse as transistors shrink
  • 3nm transistors leak significantly

This is why your phone heats up!

Speed

Transistor switching speed:

  • Modern transistors: ~10 picoseconds (0.00000000001 seconds)
  • CPU clock: 5 GHz = 0.2 nanoseconds = 200 picoseconds
  • Multiple transistors switch per clock cycle

Light travels:

  • 30 cm in 1 nanosecond
  • 6 cm in 200 picoseconds (one clock cycle)

Signals can’t travel across chip in one cycle!

  • This is why CPU design is hard
  • Pipeline, caches, prediction needed

See: CPU-Pipeline

The abstraction layers

From physics to programs:

Quantum mechanics
  ↓
Semiconductors (PN junctions)
  ↓
Transistors (switches)
  ↓
Logic gates (AND, OR, NOT)
  ↓
Combinational logic (adders, multiplexers)
  ↓
Sequential logic (flip-flops, registers)
  ↓
Functional units (ALU, memory)
  ↓
CPU (fetch-decode-execute)
  ↓
Machine code (instructions)
  ↓
Assembly language
  ↓
C/C++/Rust
  ↓
Your program

Each layer hides complexity of layer below.

This is why you can write code without understanding quantum mechanics!

Key takeaways

The fundamental insight

Computing is possible because:

  1. Electricity is fast (speed of light)
  2. We can make tiny switches (transistors)
  3. Switches can represent information (binary)
  4. Logic can be implemented with switches (gates)
  5. We can manufacture billions of them cheaply

That’s it. That’s the whole computer.

Everything is just switches

When you run a program:

  • Billions of transistors turn on/off
  • In coordinated patterns
  • Billions of times per second
  • Following simple rules (AND, OR, NOT)

The rules create:

  • Arithmetic (addition, multiplication)
  • Memory (flip-flops hold state)
  • Control flow (if/else, loops)
  • Everything your computer does

See: Computer

The miracle

Modern computers:

  • 100+ billion transistors per chip
  • 5+ GHz clock (5 billion cycles per second)
  • 3nm feature size (15 atoms wide)
  • 99.9999% reliable
  • Costs $500

If car technology improved like computers:

  • Car from 1971 cost $4,000
  • Same performance today: $0.004 (less than a penny)
  • Would go 1 million km/h
  • Would get 10,000 km per liter

Transistors are the most successful technology in human history.

The big picture

Every computation:

Your code: print("Hello")
    ↓
Compiler → Machine code
    ↓
CPU fetches instruction
    ↓
Transistors in decoder turn on/off
    ↓
More transistors fetch data from memory
    ↓
ALU transistors compute
    ↓
Output transistors send signals
    ↓
GPU transistors light pixels
    ↓
"Hello" appears on screen

All from tiny switches turning on and off.

THAT is how it knows.

The answer to “But How Do It Know?”

How does a computer know what to do?

It doesn’t.

It’s just billions of switches, following simple rules:

  • If switch A and switch B are ON, turn on switch C
  • That’s AND
  • That’s all

But from those simple rules, connected in the right way, emerges:

  • Memory
  • Arithmetic
  • Logic
  • Programs
  • Intelligence (?)

The computer doesn’t “know” anything. It’s switches all the way down.

But from switches, we built a world.
