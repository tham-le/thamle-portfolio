WTF is Encryption?

Encryption is the process of scrambling data so only authorized parties can read it. It transforms plaintext (readable) into ciphertext (unreadable gibberish).

Think of it as putting your message in a locked box - only people with the key can open it.

The Safe Analogy

The Locked Safe

Without Encryption

You write a secret message on paper and leave it on your desk. Problem: Anyone walking by can read it.

With Encryption

You write the message, put it in a safe, lock it with a combination.

  • Plaintext: The original message
  • Encryption: Locking it in the safe
  • Ciphertext: The locked safe (looks like a metal box, unreadable)
  • Decryption: Using the combination to unlock it
  • Key: The combination (e.g., ‚Äú42-17-89‚Äù)

Only people with the combination can read the message.

That‚Äôs encryption - transform data into gibberish using a key, reverse it with the same (or related) key.

Hybrid Encryption (Best of Both Worlds)

Problem: Asymmetric encryption is slow (100-1000x slower than symmetric).

Solution: Use both!

1. Generate random symmetric key (e.g., AES key)
2. Encrypt MESSAGE with symmetric key (fast!)
3. Encrypt SYMMETRIC KEY with recipient's public key (small, so fast enough)
4. Send both encrypted message + encrypted key

Receiver:

1. Decrypt symmetric key with private key
2. Decrypt message with symmetric key

This is how TLS (HTTPS) works!

Breaking Encryption

Brute force attack: Try every possible key

56-bit key: 2^56 = 72 quadrillion possibilities (doable with GPUs)
128-bit key: 2^128 = 340 undecillion possibilities (impossible)

Modern encryption is unbreakable (assuming no algorithm flaws).

Encryption in TLS

TLS (used in HTTPS) combines everything:

1. Asymmetric (RSA/ECC): Authenticate server, exchange keys
2. Symmetric (AES): Encrypt actual data
3. Hashing (SHA256): Verify integrity

Result: Secure web browsing.

The Big Takeaway

Encryption in a Nutshell

Encryption transforms readable data (plaintext) into unreadable data (ciphertext) using a key.

Two types:

  • Symmetric: Same key encrypts and decrypts (fast, key sharing problem)
  • Asymmetric: Public key encrypts, private key decrypts (slow, solves key sharing)

Real-world uses hybrid: Asymmetric to exchange symmetric key, symmetric to encrypt data.

Common algorithms:

  • AES: Symmetric, industry standard
  • RSA/ECC: Asymmetric, key exchange
  • ChaCha20-Poly1305: Modern authenticated encryption

TLS (HTTPS) uses all of these to secure web traffic.

Golden rule: Never roll your own crypto. Use proven libraries.

The rule: Encryption protects confidentiality (who can read data). Use symmetric for speed, asymmetric for key exchange, and always authenticate ciphertext to prevent tampering.
