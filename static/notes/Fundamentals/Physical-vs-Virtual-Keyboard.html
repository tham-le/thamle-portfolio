WTF is the Difference Between Physical Keyboards and Virtual Keyboards?

Or: Why your desktop sends weird codes but your phone just sends text

The Restaurant Kitchen Analogy

Physical Keyboards: The Event Stream

When you press a key on a physical keyboard, it sends a Scancode to the computer. Scan codes are hardware level identifiers for physical key positions.

How Physical Keyboards Work

Scan codes consist of a single byte where the low 7 bits identify the key, and the most significant bit is clear for a key press or set for a key release.

Here‚Äôs what happens when you press and release the ‚ÄúA‚Äù key:

Key Press:   Scancode 0x1E (00011110 in binary, bit 7 = 0)
Key Repeat:  Scancode 0x1E (sent repeatedly while held)
Key Release: Scancode 0x9E (10011110 in binary, bit 7 = 1)

Notice how key release is the same code with bit 7 flipped? Esc press produces scancode 01, Esc release produces scancode 81 (hex). The pattern is simple: add 0x80 to the press code to get the release code.

The Three Event Stream

Physical keyboards produce three types of events:

  1. Key Down (Make): Physical key pressed
  2. Key Repeat: OS generated events while key is held
  3. Key Up (Break): Physical key released

Virtual Keyboards: Direct Character Input

Virtual keyboards work completely differently. An input method editor (IME) is a user control that enables users to enter text. Android provides an extensible input method framework that allows applications to provide users alternative input methods, such as on-screen keyboards or even speech input.

What‚Äôs an IME?

An input method is an operating system component or program that enables users to generate characters not natively available on their input devices by using sequences of characters or mouse operations that are available to them.

IMEs are particularly important for languages like Chinese, Japanese, and Korean where thousands of characters can‚Äôt fit on a keyboard. The IME lets you type phonetic sequences that get converted to the actual characters you want.

But even English virtual keyboards on phones are IMEs. They just happen to be very simple ones.

The Big Picture

Here‚Äôs your mental model:

Physical Keyboards:

  • Send scan codes for physical key positions
  • Generate key down, key repeat, and key up events
  • OS translates through multiple layers to get characters
  • Perfect for games, shortcuts, and low level control
  • Same hardware, different layouts produce different characters

Virtual Keyboards (IMEs):

  • Send Unicode characters directly
  • No key up/down events, just final text
  • Handle complex input (predictions, autocorrect, gestures)
  • Perfect for text entry, especially non Latin scripts
  • What you tap is what you get (mostly)

The Core Difference

Physical keyboards say ‚Äúwhat key‚Äù, virtual keyboards say ‚Äúwhat character‚Äù. Physical keyboards report actions, virtual keyboards report results.
