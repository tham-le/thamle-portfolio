WTF is Lambda Calculus?

Lambda Calculus (λ-calculus) is a mathematical system for expressing computation using functions only. It was created by Alonzo Church in the 1930s as a way to study what can be computed.

It’s the theoretical foundation for all functional programming languages (OCaml, Haskell, Lisp, etc.).

Think of it as the simplest possible programming language - just functions, nothing else.

The Minimalist Philosophy Analogy

Building Everything from One Brick

Traditional Programming

Full toolbox: Variables, loops, objects, arrays, etc.

  • Many primitives
  • Complex language
  • Easy to write programs

Lambda Calculus

One tool: Functions

  • Only lambdas (anonymous functions)
  • Ultra-minimal
  • Surprisingly, can express everything

Like: Building an entire house using only one type of LEGO brick.

That’s lambda calculus - functions are all you need to express any computation.

The Three Rules

Lambda calculus has only three components:

1. Variables

x
y
z

Just names. Simple.

2. Abstraction (Creating Functions)

λx. x

Read as: “Lambda x. x” or “A function that takes x and returns x”

3. Application (Calling Functions)

(λx. x) y

Read as: “Apply the function (λx. x) to y”

Result: y (the identity function returns its argument)

That’s it! Everything else is built from these three things.

Examples

Identity Function

λx. x

What it does: Returns its input unchanged.

(λx. x) 5  →  5
(λx. x) hello  →  hello

Constant Function

λx. λy. x

What it does: Takes two arguments, returns the first one.

(λx. λy. x) 5 10  →  5

Application:

(λx. λy. x) 5      →  λy. 5      (first argument captured)
(λy. 5) 10         →  5          (second argument ignored)

Self-Application

λx. (x x)

What it does: Applies a function to itself.

Careful! This can create infinite loops:

(λx. (x x)) (λx. (x x))  →  (λx. (x x)) (λx. (x x))  →  ...

This is the famous omega combinator (never terminates).

Encoding Everything with Functions

Booleans

TRUE  = λx. λy. x     (Returns first argument)
FALSE = λx. λy. y     (Returns second argument)

IF = λcond. λthen. λelse. cond then else

IF TRUE 5 10   →  5
IF FALSE 5 10  →  10

Numbers (Church Numerals)

0 = λf. λx. x              (Apply f zero times)
1 = λf. λx. f x            (Apply f once)
2 = λf. λx. f (f x)        (Apply f twice)
3 = λf. λx. f (f (f x))    (Apply f three times)

Successor (n + 1):

SUCC = λn. λf. λx. f (n f x)

SUCC 2  →  3

Addition:

ADD = λm. λn. λf. λx. m f (n f x)

ADD 2 3  →  5

Multiplication:

MULT = λm. λn. λf. m (n f)

MULT 2 3  →  6

You can encode arithmetic using only functions!

Pairs (Tuples)

PAIR = λx. λy. λf. f x y
FIRST = λp. p (λx. λy. x)
SECOND = λp. p (λx. λy. y)

PAIR 5 10           →  λf. f 5 10
FIRST (PAIR 5 10)   →  5
SECOND (PAIR 5 10)  →  10

Lists

NIL = λx. TRUE            (Empty list)
CONS = λh. λt. λf. f h t  (Prepend element)
HEAD = λl. l (λh. λt. h)
TAIL = λl. l (λh. λt. t)

CONS 1 NIL              →  [1]
CONS 2 (CONS 1 NIL)     →  [2, 1]
HEAD (CONS 2 ...)       →  2

Recursion with the Y Combinator

Problem: Lambda calculus has no loops or recursion built-in.

Solution: The Y combinator enables recursion:

Y = λf. (λx. f (x x)) (λx. f (x x))

Example: Factorial

The Y combinator lets you define recursive functions without naming them!

Beta Reduction (Evaluation)

Beta reduction is how you “run” lambda calculus:

(λx. x + 1) 5
→ 5 + 1
→ 6

Steps:

  1. Replace parameter (x) with argument (5)
  2. Evaluate expression

Example:

(λx. λy. x + y) 3 4
→ (λy. 3 + y) 4
→ 3 + 4
→ 7

Why Lambda Calculus Matters

1. Theoretical Foundation

Lambda calculus is Turing-complete - it can compute anything a Turing machine can.

Church-Turing Thesis: Anything computable can be computed with lambda calculus.

2. Programming Language Design

Languages influenced by lambda calculus:

  • Lisp (1958) - First lambda-based language
  • ML / OCaml (1973) - Strong types + lambdas
  • Haskell (1990) - Pure functional
  • JavaScript (1995) - First-class functions
  • Rust (2010) - Closures inspired by lambda calculus

3. Type Systems

Simply Typed Lambda Calculus adds types:

λx:Int. x + 1  :  Int → Int

This is the foundation of Compiler type systems.

4. Functional Programming

Core concepts from lambda calculus:

  • First-class functions
  • Higher-order functions (functions that take/return functions)
  • Closures (functions that capture their environment)
  • Immutability

The Big Takeaway

Lambda Calculus in a Nutshell

Lambda Calculus is a minimal mathematical system for computation using only functions.

Three components:

  1. Variables - x, y, z
  2. Abstraction - λx. x (create function)
  3. Application - (λx. x) y (call function)

You can encode:

  • Numbers, booleans, pairs, lists
  • Arithmetic operations
  • Recursion (Y combinator)
  • Anything computable!

Influenced:

  • Functional programming languages (OCaml, Haskell, Lisp)
  • First-class functions in JavaScript, Python, Rust
  • Type systems in Compilers

Key insight: Functions are universal - you can build everything from them.

The rule: Lambda calculus proves that functions are enough to express any computation. Every time you write (x) => x + 1 in JavaScript or |x| x + 1 in Rust, you’re using ideas from lambda calculus. It’s the theoretical foundation of functional programming.
