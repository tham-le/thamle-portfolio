Data structures are the foundation of efficient algorithms. Choose the right structure, and your code flies. Choose wrong, and it crawls.

This guide covers 8 fundamental data structures every developer must know.

Why Data Structures Matter

The Performance Impact

Same problem, different structures:

Find if element exists in 1,000,000 items:

  • Array (unsorted): 1,000,000 operations → 1 second
  • Sorted Array + Binary Search: 20 operations → 0.00002 seconds
  • Hash Table: 1 operation → 0.000001 seconds

50,000x faster just by choosing the right structure!

1. Arrays

What It Is

Contiguous block of memory storing elements of the same type.

Visual Representation

Index:  0    1    2    3    4
Value: [10] [20] [30] [40] [50]
       ↑
    Address: 1000

Operations Complexity

OperationTime Complexity
Access by indexO(1)
Search (unsorted)O(n)
Search (sorted)O(log n)
Insert at endO(1) amortized
Insert at beginningO(n)
DeleteO(n)

Implementation

When to Use

  • ✅ Need fast random access by index
  • ✅ Know size in advance or size doesn’t change much
  • ✅ Memory locality is important (cache-friendly)
  • ❌ Frequent insertions/deletions in middle

Real-World Uses

  • Image pixels (2D array)
  • Game grids (chess board, tic-tac-toe)
  • Buffer for streaming data
  • Database table rows

2. Linked Lists

What It Is

Sequence of nodes, each containing data and pointer to next node.

Visual Representation

Singly Linked List:
Head → [10|•] → [20|•] → [30|•] → [40|∅]

Doubly Linked List:
Head → [∅|10|•] ⇄ [•|20|•] ⇄ [•|30|•] ⇄ [•|40|∅]

Operations Complexity

OperationTime Complexity
Access by indexO(n)
SearchO(n)
Insert at beginningO(1)
Insert at endO(1) with tail pointer
Delete (given node)O(1)
Delete (by value)O(n)

Implementation

When to Use

  • ✅ Frequent insertions/deletions at beginning
  • ✅ Don’t know size in advance
  • ✅ Don’t need random access
  • ❌ Need to access by index frequently

Real-World Uses

  • Music playlist (next/previous)
  • Browser history (back/forward)
  • Undo/redo functionality
  • Hash table collision handling (chaining)

Visual Representation

        ┌─────┐
push →  │  30 │  ← top
        ├─────┤
        │  20 │
        ├─────┤
        │  10 │  ← bottom
        └─────┘

Operations Complexity

OperationTime Complexity
PushO(1)
PopO(1)
PeekO(1)
SearchO(n)

Implementation

Common Stack Problems

When to Use

  • ✅ Need to reverse order
  • ✅ Track history (undo/redo)
  • ✅ Expression evaluation
  • ✅ Backtracking algorithms

Real-World Uses

  • Function call stack
  • Browser back button
  • Text editor undo
  • Expression parsing
  • Depth-First Search (DFS)

Visual Representation

Front                          Rear
  ↓                             ↓
[10] → [20] → [30] → [40] → [50]
dequeue ←              → enqueue

Operations Complexity

OperationTime Complexity
EnqueueO(1)
DequeueO(1)
PeekO(1)
SearchO(n)

Implementation

Common Queue Problems

When to Use

  • ✅ Process items in order received
  • ✅ BFS traversal
  • ✅ Task scheduling
  • ✅ Buffering

Real-World Uses

  • Print job queue
  • CPU task scheduling
  • Breadth-First Search (BFS)
  • Message queues (RabbitMQ, Kafka)
  • Call center wait lines

5. Hash Tables

What It Is

Key-value pairs with O(1) average lookup time.

Visual Representation

Hash Function: hash(key) % table_size

Keys: ["apple", "banana", "orange"]

    0: → ["banana", 2] → ["orange", 3]  (collision chain)
    1: empty
    2: → ["apple", 1]
    3: empty
    4: empty

Operations Complexity

OperationAverageWorst Case
InsertO(1)O(n)
DeleteO(1)O(n)
SearchO(1)O(n)

Implementation

Common Hash Table Problems

When to Use

  • ✅ Need fast lookups by key
  • ✅ Counting/frequency problems
  • ✅ Caching
  • ✅ Removing duplicates

Real-World Uses

  • Database indexing
  • Caching (Redis, Memcached)
  • Symbol tables in compilers
  • DNS lookup
  • Password verification

6. Trees

What It Is

Hierarchical structure with root and child nodes.

Visual Representation

Binary Tree:
        10
       /  \
      5    15
     / \   / \
    3   7 12  20

Binary Search Tree (BST):
- Left subtree < Node
- Right subtree > Node

Operations Complexity (BST)

OperationAverageWorst Case
SearchO(log n)O(n)
InsertO(log n)O(n)
DeleteO(log n)O(n)

Implementation

Tree Traversals

When to Use

  • ✅ Hierarchical data
  • ✅ Fast search in sorted data
  • ✅ Range queries
  • ✅ Prefix matching

Real-World Uses

  • File systems
  • DOM tree (HTML)
  • Database indexes (B-trees)
  • Expression parsing
  • Autocomplete (Trie)

7. Heaps

What It Is

Complete binary tree satisfying heap property.

Min Heap: Parent ≤ Children Max Heap: Parent ≥ Children

Visual Representation

Min Heap:
        1
       / \
      3   2
     / \ / \
    7  5 4  6

Array: [1, 3, 2, 7, 5, 4, 6]
Parent of i: (i-1)//2
Left child: 2i + 1
Right child: 2i + 2

Operations Complexity

OperationTime Complexity
Find min/maxO(1)
InsertO(log n)
Delete min/maxO(log n)
HeapifyO(n)

Implementation

Common Heap Problems

When to Use

  • ✅ Need min/max quickly
  • ✅ Priority queue
  • ✅ Top K elements
  • ✅ Merge K sorted arrays

Real-World Uses

  • Priority scheduling
  • Dijkstra’s algorithm
  • Huffman encoding
  • Event-driven simulation
  • Top K elements (trending topics)

Decision Tree: Which Structure?

Need to...
│
├─ Store ordered collection?
│  ├─ Need random access? → Array
│  └─ Frequent insert/delete? → Linked List
│
├─ Last in, first out? → Stack
├─ First in, first out? → Queue
│
├─ Fast lookup by key? → Hash Table
│
├─ Maintain sorted order?
│  ├─ Need min/max quickly? → Heap
│  └─ Need all operations fast? → Balanced BST
│
├─ Model relationships? → Graph
│
└─ Hierarchical data? → Tree

Key Takeaways

Choose the right tool for the job:

  • Arrays: Random access, fixed size
  • Linked Lists: Dynamic size, frequent insert/delete
  • Stacks: LIFO, undo/redo, DFS
  • Queues: FIFO, BFS, scheduling
  • Hash Tables: Fast lookups, counting
  • Trees: Hierarchical, sorted data
  • Heaps: Priority queue, top K
  • Graphs: Relationships, networks

Master these 8, and you can solve almost any problem efficiently.
