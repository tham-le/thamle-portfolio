Sorting Algorithms Complete Guide

Sorting Algorithms Complete Guide

Nov 07, 202515 min read

  • dsa

  • algorithms
  • implementation

Sorting is fundamental. It’s the first algorithm most developers learn and one of the most important.

This guide covers 10 essential sorting algorithms from basic to advanced.

Why Sorting Matters

The Power of Sorted Data

Unsorted array: Search takes O(n) Sorted array: Binary search takes O(log n)

For 1,000,000 elements:

  • Unsorted: 1,000,000 comparisons
  • Sorted: ~20 comparisons

50,000x faster searches after sorting!

Real-World Applications

  • Databases: Index sorting for fast queries
  • E-commerce: Sort products by price, rating
  • Social media: Sort posts by time, relevance
  • File systems: Sort files by name, date, size
  • Search engines: Rank results

1. Bubble Sort

How It Works

Repeatedly compare adjacent elements and swap if wrong order. Largest element “bubbles” to the end each pass.

Visual Example

Pass 1: [5, 3, 8, 4, 2]
        [3, 5, 8, 4, 2]  (swap 5,3)
        [3, 5, 4, 8, 2]  (swap 8,4)
        [3, 5, 4, 2, 8]  (swap 8,2) ← 8 in place

Pass 2: [3, 5, 4, 2, 8]
        [3, 4, 5, 2, 8]
        [3, 4, 2, 5, 8] ← 5 in place

Pass 3: [3, 4, 2, 5, 8]
        [3, 2, 4, 5, 8]
        [3, 2, 4, 5, 8] ← 4 in place

Pass 4: [2, 3, 4, 5, 8] ← Sorted!

Implementation

Complexity Analysis

  • Time: O(n²) average and worst, O(n) best (already sorted)
  • Space: O(1)
  • Stable: Yes

When to Use

  • ✅ Small datasets (<10 elements)
  • ✅ Nearly sorted data
  • ✅ Teaching purposes
  • ❌ Large datasets (very slow)

2. Selection Sort

How It Works

Find minimum element, swap with first position. Repeat for remaining unsorted portion.

Visual Example

[64, 25, 12, 22, 11]
 ↑              ↓
Find min (11), swap with first:
[11, 25, 12, 22, 64]
     ↑       ↓
Find min (12), swap:
[11, 12, 25, 22, 64]
         ↑   ↓
Find min (22), swap:
[11, 12, 22, 25, 64]
             ↑   ↓
Find min (25), already in place:
[11, 12, 22, 25, 64] ← Sorted!

Implementation

Complexity Analysis

  • Time: O(n²) all cases
  • Space: O(1)
  • Stable: No (can be made stable with extra space)

When to Use

  • ✅ Small datasets
  • ✅ Memory is very limited (in-place)
  • ❌ Need stability
  • ❌ Large datasets

3. Insertion Sort

How It Works

Build sorted array one element at a time by inserting each element into its correct position.

Visual Example

[5, 2, 4, 6, 1, 3]

Start: [5] | 2, 4, 6, 1, 3
Insert 2: [2, 5] | 4, 6, 1, 3
Insert 4: [2, 4, 5] | 6, 1, 3
Insert 6: [2, 4, 5, 6] | 1, 3
Insert 1: [1, 2, 4, 5, 6] | 3
Insert 3: [1, 2, 3, 4, 5, 6] ← Sorted!

Implementation

Complexity Analysis

  • Time: O(n²) average/worst, O(n) best (already sorted)
  • Space: O(1)
  • Stable: Yes

When to Use

  • ✅ Small datasets
  • ✅ Nearly sorted data (very fast!)
  • ✅ Online algorithm (sort as data arrives)
  • ✅ Simple implementation needed

Real-World Use

  • Used in Timsort for small subarrays
  • Sorting playing cards
  • Incremental sorting

4. Merge Sort

How It Works

Divide array in half recursively, sort each half, merge sorted halves.

Visual Example

[38, 27, 43, 3, 9, 82, 10]

Divide:
[38, 27, 43, 3] [9, 82, 10]
[38, 27] [43, 3] [9, 82] [10]
[38] [27] [43] [3] [9] [82] [10]

Conquer (Merge):
[27, 38] [3, 43] [9, 82] [10]
[3, 27, 38, 43] [9, 10, 82]
[3, 9, 10, 27, 38, 43, 82] ← Sorted!

Implementation

In-Place Merge Sort (Space Optimized)

Complexity Analysis

  • Time: O(n log n) all cases
  • Space: O(n)
  • Stable: Yes

When to Use

  • ✅ Need guaranteed O(n log n)
  • ✅ Stability required
  • ✅ Sorting linked lists (O(1) space)
  • ✅ External sorting (large files)
  • ❌ Space is very limited

Real-World Use

  • Database sorting
  • External sorting (files too large for memory)
  • Python’s sorted() partially uses merge sort

5. Quick Sort

How It Works

Pick pivot, partition array (smaller left, larger right), recursively sort partitions.

Visual Example

[10, 80, 30, 90, 40, 50, 70]
Pivot = 70

Partition:
[10, 30, 40, 50] [70] [80, 90]

Recursively sort:
[10, 30, 40, 50] → [10] [30, 40, 50]
[30, 40, 50] → [30] [40, 50]
...

Result: [10, 30, 40, 50, 70, 80, 90]

Implementation

Pivot Selection Strategies

Complexity Analysis

  • Time: O(n log n) average, O(n²) worst
  • Space: O(log n) for recursion stack
  • Stable: No

When to Use

  • ✅ General purpose sorting (fastest average case)
  • ✅ Good cache performance
  • ✅ In-place sorting needed
  • ❌ Need guaranteed O(n log n)
  • ❌ Need stability

Real-World Use

  • C’s qsort(), most standard libraries
  • Default sorting in many languages
  • Used when average case performance matters

6. Heap Sort

How It Works

Build max heap, repeatedly extract maximum and rebuild heap.

Visual Example

[4, 10, 3, 5, 1]

Build Max Heap:
        10
       /  \
      5    3
     / \
    4   1

Extract max (10): [1, 5, 3, 4] | 10
Heapify: [5, 4, 3, 1] | 10
Extract max (5): [1, 4, 3] | 5, 10
Heapify: [4, 1, 3] | 5, 10
...
Result: [1, 3, 4, 5, 10]

Implementation

Complexity Analysis

  • Time: O(n log n) all cases
  • Space: O(1)
  • Stable: No

When to Use

  • ✅ Need guaranteed O(n log n) with O(1) space
  • ✅ No extra memory available
  • ❌ Need stability
  • ❌ Need best cache performance

7. Counting Sort

How It Works

Count occurrences of each value, reconstruct sorted array from counts.

Restriction: Works only for integers in known range.

Visual Example

Input: [4, 2, 2, 8, 3, 3, 1]
Range: 1-8

Count array:
Index: 1  2  3  4  5  6  7  8
Count: 1  2  2  1  0  0  0  1

Cumulative count:
Index: 1  2  3  4  5  6  7  8
Count: 1  3  5  6  6  6  6  7

Output: [1, 2, 2, 3, 3, 4, 8]

Implementation

Complexity Analysis

  • Time: O(n + k) where k is range
  • Space: O(k)
  • Stable: Yes

When to Use

  • ✅ Small range of integers (k ≈ n)
  • ✅ Need linear time sorting
  • ✅ Counting frequency anyway
  • ❌ Large range (memory expensive)
  • ❌ Non-integer data

Real-World Use

  • Sorting ages (0-120)
  • Sorting grades (0-100)
  • Sorting small integers
  • Used in Radix Sort

8. Radix Sort

How It Works

Sort digit by digit using counting sort, from least to most significant digit.

Visual Example

Input: [170, 45, 75, 90, 802, 24, 2, 66]

Sort by 1s place:
[170, 90, 802, 2, 24, 45, 75, 66]

Sort by 10s place:
[802, 2, 24, 45, 66, 170, 75, 90]

Sort by 100s place:
[2, 24, 45, 66, 75, 90, 170, 802] ← Sorted!

Implementation

Complexity Analysis

  • Time: O(d × n) where d is number of digits
  • Space: O(n + k)
  • Stable: Yes

When to Use

  • ✅ Integers with fixed number of digits
  • ✅ Need linear time for large datasets
  • ✅ Strings of same length
  • ❌ Floating-point numbers
  • ❌ Variable-length data

Real-World Use

  • Sorting phone numbers
  • Sorting ZIP codes
  • Sorting fixed-width integers
  • Parallel sorting (GPU)

9. Bucket Sort

How It Works

Distribute elements into buckets, sort each bucket, concatenate.

Visual Example

Input: [0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12]
Range: [0, 1)
Buckets: 5

Bucket 0 [0.0-0.2): [0.17, 0.12]
Bucket 1 [0.2-0.4): [0.39, 0.26, 0.21]
Bucket 2 [0.4-0.6): []
Bucket 3 [0.6-0.8): [0.78, 0.72]
Bucket 4 [0.8-1.0): [0.94]

Sort each bucket (insertion sort):
Bucket 0: [0.12, 0.17]
Bucket 1: [0.21, 0.26, 0.39]
Bucket 2: []
Bucket 3: [0.72, 0.78]
Bucket 4: [0.94]

Concatenate:
[0.12, 0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94]

Implementation

Complexity Analysis

  • Time: O(n + k) average, O(n²) worst
  • Space: O(n + k)
  • Stable: Yes (if stable sort used in buckets)

When to Use

  • ✅ Uniformly distributed data
  • ✅ Floating-point numbers
  • ✅ Know data range in advance
  • ❌ Data is not uniformly distributed

Real-World Use

  • Sorting floating-point numbers
  • External sorting
  • Parallel sorting

10. Tim Sort

How It Works

Hybrid of Merge Sort and Insertion Sort. Python’s default sorting algorithm.

Strategy:

  1. Divide array into small “runs” (32-64 elements)
  2. Sort each run with Insertion Sort (fast for small arrays)
  3. Merge runs using Merge Sort

Why It’s Fast

  • Takes advantage of already sorted data
  • Insertion sort fast for small runs
  • Merge sort stable and efficient for larger data
  • Best of both worlds

Simplified Implementation

Complexity Analysis

  • Time: O(n) best, O(n log n) average/worst
  • Space: O(n)
  • Stable: Yes

When to Use

  • ✅ Already used in Python and Java!
  • ✅ Real-world data (often partially sorted)
  • ✅ Need stability
  • ✅ General-purpose sorting

Real-World Use

  • Python’s sorted() and list.sort()
  • Java’s Arrays.sort() (for objects)
  • Most production systems

Which Sort to Use?

Decision Tree

Need to sort?
│
├─ Small array (<10-50 elements)?
│  └─ Insertion Sort
│
├─ Nearly sorted data?
│  └─ Insertion Sort or Tim Sort
│
├─ Need guaranteed O(n log n)?
│  ├─ Memory available? → Merge Sort
│  └─ No extra memory? → Heap Sort
│
├─ General purpose?
│  ├─ Need stability? → Merge Sort or Tim Sort
│  └─ Don't need stability? → Quick Sort
│
├─ Integers in small range?
│  └─ Counting Sort
│
├─ Fixed-digit integers?
│  └─ Radix Sort
│
└─ Uniformly distributed floats?
   └─ Bucket Sort

Quick Reference

For most cases: Use Quick Sort (fast average) or Tim Sort (stable, adaptive)

For specific scenarios:

  • Small data: Insertion Sort
  • Guaranteed O(n log n): Merge Sort or Heap Sort
  • Stability required: Merge Sort, Tim Sort, or Counting Sort
  • Integer range known: Counting Sort or Radix Sort
  • No extra memory: Heap Sort or Quick Sort
  • Already sorted: Tim Sort or Insertion Sort

Stability Example

Why does stability matter?

Sort by score, then by name:

Before:
Alice 85
Bob 90
Charlie 85
Dave 90

After (unstable sort by score):
Bob 90     ← Could be
Dave 90    ← Flipped!
Charlie 85
Alice 85

After (stable sort by score):
Bob 90     ← Original order
Dave 90    ← Preserved!
Alice 85
Charlie 85

Stable sorts: Merge, Insertion, Bubble, Tim, Counting, Radix Unstable sorts: Quick, Heap, Selection

Key Takeaways

Sorting is fundamental:

  • Quick Sort: Fastest average case
  • Merge Sort: Stable, guaranteed O(n log n)
  • Heap Sort: In-place, guaranteed O(n log n)
  • Tim Sort: Best for real-world data (Python/Java default)
  • Counting/Radix: Linear time for integers

Choose based on:

  • Data size
  • Memory constraints
  • Stability requirements
  • Data distribution
  • Expected case (best/average/worst)
