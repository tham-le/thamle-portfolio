Data Structures & Algorithms Roadmap

A systematic approach to learning DSA, organized by difficulty and dependencies.

üìã Prerequisites

Before starting DSA:

  • Comfortable with a programming language (C, C++, Python, or Java)
  • Basic understanding of Memory and Pointers (for C/C++)
  • Familiar with recursion
  • Know basic math (logarithms, exponents, combinatorics)

üéØ Learning Philosophy

1. Understand the ‚ÄúWhy‚Äù

Don‚Äôt just memorize - understand when and why to use each structure.

2. Implement from Scratch

Build each data structure yourself before using library implementations.

3. Practice Deliberately

Quality > quantity. Understand each problem deeply.

4. Time Complexity First

Always analyze time/space complexity before coding.

üìö Phase 1: Foundations (2-4 weeks)

Big-O Notation & Complexity Analysis

  • Time complexity (O(1), O(log n), O(n), O(n log n), O(n¬≤))
  • Space complexity
  • Best/average/worst case
  • Amortized analysis

Practice:

  • Analyze loops, nested loops
  • Compare algorithms

Resources:

  • Big-O Cheat Sheet

Arrays

  • Fixed-size arrays
  • Dynamic arrays (vectors)
  • 2D arrays
  • String manipulation

Key operations:

  • Access: O(1)
  • Search: O(n)
  • Insert/Delete: O(n)

Practice problems:

  • Two Sum
  • Remove Duplicates
  • Rotate Array
  • Maximum Subarray

Linked Lists

  • Singly linked list
  • Doubly linked list
  • Circular linked list

Key operations:

  • Access: O(n)
  • Insert/Delete at head: O(1)
  • Insert/Delete at tail: O(n) (O(1) with tail pointer)

Practice problems:

  • Reverse Linked List
  • Detect Cycle
  • Merge Two Sorted Lists
  • Remove Nth Node From End

See also: Linked-List (create this!)

Stacks & Queues

  • Stack (LIFO)
  • Queue (FIFO)
  • Deque (double-ended queue)
  • Priority Queue (heap-based)

Applications:

  • Stack: Function calls, undo/redo, expression evaluation
  • Queue: BFS, task scheduling

Practice problems:

  • Valid Parentheses
  • Min Stack
  • Implement Queue using Stacks
  • Sliding Window Maximum

See also: Stack-and-Queue (create this!)

üìö Phase 2: Trees & Recursion (3-5 weeks)

Binary Trees

  • Tree terminology (root, leaf, height, depth)
  • Tree traversals (inorder, preorder, postorder, level-order)
  • Binary Search Tree (BST)
  • Balanced trees (AVL, Red-Black)

Key properties (BST):

  • Left subtree < root < right subtree
  • Search/Insert/Delete: O(log n) average, O(n) worst

Practice problems:

  • Maximum Depth of Binary Tree
  • Validate BST
  • Lowest Common Ancestor
  • Serialize and Deserialize Binary Tree

See also: Binary-Tree (create this!)

Heaps

  • Min heap / Max heap
  • Heap operations (insert, extract-min, heapify)
  • Priority queue implementation

Key operations:

  • Insert: O(log n)
  • Extract-min: O(log n)
  • Peek: O(1)

Applications:

  • Dijkstra‚Äôs algorithm
  • Huffman coding
  • Finding k-th largest element

Practice problems:

  • Kth Largest Element
  • Merge K Sorted Lists
  • Top K Frequent Elements

See also: Heap (create this!)

Advanced Trees

  • Trie (prefix tree)
  • Segment tree
  • Fenwick tree (Binary Indexed Tree)

Applications:

  • Trie: Autocomplete, spell checker
  • Segment tree: Range queries
  • Fenwick tree: Cumulative frequency

üìö Phase 3: Graphs (4-6 weeks)

Graph Basics

  • Graph representations (adjacency list, matrix)
  • Directed vs undirected
  • Weighted vs unweighted
  • Degree, path, cycle, connectivity

Traversals:

  • DFS (Depth-First Search) - Stack/Recursion
  • BFS (Breadth-First Search) - Queue

Practice problems:

  • Number of Islands
  • Clone Graph
  • Course Schedule (topological sort)
  • Network Delay Time

See also: Graph (create this!)

Shortest Path Algorithms

  • Dijkstra‚Äôs algorithm (single source, non-negative)
  • Bellman-Ford (single source, negative edges)
  • Floyd-Warshall (all pairs)
  • A* (heuristic-based)

Time complexity:

  • Dijkstra: O((V + E) log V) with min-heap
  • Bellman-Ford: O(VE)
  • Floyd-Warshall: O(V¬≥)

Advanced Graph Algorithms

  • Topological sort
  • Strongly connected components (Tarjan‚Äôs/Kosaraju‚Äôs)
  • Minimum spanning tree (Kruskal‚Äôs/Prim‚Äôs)
  • Network flow (Ford-Fulkerson)

üìö Phase 4: Algorithms (4-6 weeks)

Sorting Algorithms

  • Comparison sorts
    • Bubble sort: O(n¬≤)
    • Selection sort: O(n¬≤)
    • Insertion sort: O(n¬≤)
    • Merge sort: O(n log n)
    • Quick sort: O(n log n) average, O(n¬≤) worst
    • Heap sort: O(n log n)
  • Non-comparison sorts
    • Counting sort: O(n + k)
    • Radix sort: O(d * (n + k))
    • Bucket sort: O(n + k)

When to use what?

  • Quick sort: General purpose, in-place
  • Merge sort: Stable, guaranteed O(n log n)
  • Heap sort: In-place, guaranteed O(n log n)

See also: Sorting-Algorithms (create this!)

Searching Algorithms

  • Linear search: O(n)
  • Binary search: O(log n)
  • Ternary search
  • Exponential search

Binary search variants:

  • First/last occurrence
  • Search in rotated array
  • Search in 2D matrix

Dynamic Programming

Key concept: Break problems into overlapping subproblems.

Patterns:

  • 1D DP (Fibonacci, climbing stairs)
  • 2D DP (longest common subsequence, edit distance)
  • Knapsack problems
  • DP on trees
  • DP with bitmasks

Approach:

  1. Define state
  2. Find recurrence relation
  3. Identify base cases
  4. Determine evaluation order

Classic problems:

  • Fibonacci
  • Coin Change
  • Longest Increasing Subsequence
  • 0/1 Knapsack
  • Edit Distance
  • Longest Common Subsequence

See also: Dynamic-Programming (create this!)

Greedy Algorithms

Key concept: Make locally optimal choice at each step.

When greedy works:

  • Optimal substructure
  • Greedy choice property

Classic problems:

  • Activity Selection
  • Huffman Coding
  • Fractional Knapsack
  • Minimum Spanning Tree

Backtracking

Key concept: Try all possibilities, backtrack on failure.

Template:

Classic problems:

  • N-Queens
  • Sudoku Solver
  • Generate Parentheses
  • Subsets / Permutations / Combinations

üéØ Study Strategy

Daily Routine

Week 1-4:   1 concept + 2-3 problems/day
Week 5-12:  2-3 problems/day (varying difficulty)
Week 13+:   Daily practice, weekly contests

Problem-Solving Approach

  1. Understand - Read carefully, ask clarifying questions
  2. Plan - Identify data structure/algorithm, draw diagrams
  3. Analyze - Determine time/space complexity BEFORE coding
  4. Code - Clean, readable implementation
  5. Test - Edge cases, performance
  6. Review - Learn from mistakes, optimize

When Stuck

  1. Try for 30 minutes
  2. Look at hints (not solution)
  3. If still stuck, read solution
  4. Recode from scratch (don‚Äôt copy-paste)
  5. Revisit problem after 1 week

üí° Tips

  1. Focus on understanding, not memorization
  2. Write clean code - Practice for interviews
  3. Explain solutions aloud - Improves clarity
  4. Review past problems - Spaced repetition
  5. Join study groups - Learn from others
  6. Track progress - Motivation boost

üö´ Common Pitfalls

  1. Jumping to hard problems too early - Build foundations first
  2. Not analyzing complexity - Always calculate Big-O
  3. Copying solutions - You won‚Äôt learn
  4. Giving up too quickly - Struggle is learning
