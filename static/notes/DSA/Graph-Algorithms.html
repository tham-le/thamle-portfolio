Graph Algorithms Deep Dive

Graphs model the connected world. Social networks, maps, dependencies, networks—all are graphs.

This guide covers 10 essential graph algorithms every developer must know.

What Are Graphs?

Definition

A graph G = (V, E) consists of:

  • V: Set of vertices (nodes)
  • E: Set of edges (connections)

Types of Graphs

1. Directed vs Undirected

Undirected:          Directed (Digraph):
    A --- B              A → B
    |     |              ↓   ↓
    C --- D              C ← D

2. Weighted vs Unweighted

Unweighted:          Weighted:
    A --- B              A --5-- B
    |     |              |       |
    C --- D              2       3
                         |       |
                         C --1-- D

3. Cyclic vs Acyclic

Cyclic:              Acyclic (DAG):
    A → B                A → B
    ↓   ↓                ↓   ↓
    C ← D                C   D

Graph Representations

1. Adjacency List

Best for sparse graphs

2. Adjacency Matrix

Best for dense graphs

3. Edge List

Simple but inefficient

1. Breadth-First Search (BFS)

What It Does

Explores graph level by level, visiting all neighbors before going deeper.

Visual Example

        A
       / \
      B   C
     / \   \
    D   E   F

BFS from A:
Level 0: [A]
Level 1: [B, C]
Level 2: [D, E, F]

Order: A → B → C → D → E → F

Implementation

Applications

Complexity

  • Time: O(V + E)
  • Space: O(V)

When to Use

  • ✅ Shortest path (unweighted)
  • ✅ Level-by-level exploration
  • ✅ Check connectivity
  • ✅ Minimum spanning tree (Prim’s)

2. Depth-First Search (DFS)

What It Does

Explores as far as possible along each branch before backtracking.

Visual Example

        A
       / \
      B   C
     / \   \
    D   E   F

DFS from A (preorder):
A → B → D → E → C → F

Implementation

Applications

Complexity

  • Time: O(V + E)
  • Space: O(V) for recursion stack

When to Use

  • ✅ Detect cycles
  • ✅ Topological sort
  • ✅ Find strongly connected components
  • ✅ Solve mazes

3. Dijkstra’s Algorithm

What It Does

Finds shortest path from source to all vertices in weighted graph (non-negative weights).

Visual Example

        A
      2/ \3
      B   C
     1\  /2
       D

Shortest paths from A:
A → A: 0
A → B: 2
A → C: 3
A → D: 3 (via B)

Implementation

Example Usage

Complexity

  • Time: O((V + E) log V) with min-heap
  • Space: O(V)

When to Use

  • ✅ Shortest path in weighted graph
  • ✅ Non-negative weights only
  • ✅ Single source to all destinations
  • ❌ Negative weights (use Bellman-Ford)

Real-World Uses

  • GPS navigation
  • Network routing
  • Flight routes

4. Bellman-Ford Algorithm

What It Does

Finds shortest path from source, handles negative weights, detects negative cycles.

Algorithm

Relax all edges V-1 times. If still updating on Vth iteration, negative cycle exists.

Implementation

Complexity

  • Time: O(V × E)
  • Space: O(V)

When to Use

  • ✅ Graph has negative weights
  • ✅ Need to detect negative cycles
  • ❌ Only positive weights (Dijkstra faster)

5. Topological Sort

What It Does

Linear ordering of vertices in DAG such that for every edge (u, v), u comes before v.

Visual Example

   A → B → D
   ↓   ↓
   C → E

Topological orders:
- A, B, C, D, E
- A, C, B, D, E
- A, C, B, E, D

Implementation

Applications

Complexity

  • Time: O(V + E)
  • Space: O(V)

When to Use

  • ✅ Task scheduling with dependencies
  • ✅ Build systems
  • ✅ Course prerequisites
  • ✅ Compiler symbol resolution

Real-World Uses

  • Build tools (Make, Gradle)
  • Course scheduling
  • Package managers (npm, pip)

6. Minimum Spanning Tree (MST)

What It Is

Subset of edges connecting all vertices with minimum total weight (no cycles).

Prim’s Algorithm

Grow MST from starting vertex, always add cheapest edge to tree.

Kruskal’s Algorithm

Sort all edges, add cheapest edge that doesn’t create cycle (Union-Find).

Complexity

  • Prim’s: O(E log V)
  • Kruskal’s: O(E log E)

When to Use

  • ✅ Network design (minimize cable)
  • ✅ Clustering
  • ✅ Approximation for TSP

Real-World Uses

  • Network cable layout
  • Circuit design
  • Clustering algorithms

7. Floyd-Warshall Algorithm

What It Does

Finds shortest paths between all pairs of vertices. Handles negative weights.

Implementation

Complexity

  • Time: O(V³)
  • Space: O(V²)

When to Use

  • ✅ Need all-pairs shortest paths
  • ✅ Dense graphs
  • ✅ Small graphs (V < 400)
  • ❌ Large graphs (too slow)

8. Union-Find (Disjoint Set)

What It Does

Track connected components, union operations, check connectivity.

Implementation

Applications

Complexity

  • Find: O(α(n)) ≈ O(1) amortized
  • Union: O(α(n)) ≈ O(1) amortized

α(n) is inverse Ackermann function, practically constant.

When to Use

  • ✅ Dynamic connectivity
  • ✅ Kruskal’s MST
  • ✅ Detect cycles
  • ✅ Connected components

9. Strongly Connected Components (SCC)

What It Is

In directed graph, maximal set of vertices where every vertex is reachable from every other.

Kosaraju’s Algorithm

Complexity

  • Time: O(V + E)
  • Space: O(V)

When to Use

  • ✅ Analyze directed graphs
  • ✅ Find mutually reachable nodes
  • ✅ Compiler optimization

10. A* Search Algorithm

What It Does

Finds shortest path using heuristic to guide search (informed search).

Implementation

Complexity

  • Time: O(E) with good heuristic
  • Space: O(V)

When to Use

  • ✅ Pathfinding with spatial information
  • ✅ Game AI
  • ✅ GPS navigation
  • ✅ Puzzle solving

Real-World Uses

  • Game pathfinding
  • GPS routing
  • Robotics

Key Takeaways

Graph algorithms power the connected world:

  • BFS: Shortest path (unweighted), level-order
  • DFS: Detect cycles, topological sort
  • Dijkstra: Shortest path (weighted, non-negative)
  • Bellman-Ford: Shortest path (negative weights)
  • Floyd-Warshall: All-pairs shortest paths
  • MST: Prim’s (dense), Kruskal’s (sparse)
  • Union-Find: Dynamic connectivity
  • A:* Informed pathfinding

Master these, and you can model and solve complex real-world problems.
