DSA Pattern Recognition Guide

Problem-solving is pattern matching. Once you recognize the pattern, you know the approach.

This guide covers 14 fundamental patterns that solve 90% of coding interview problems.

Why Pattern Recognition?

The Power of Patterns

Without patterns: Every problem feels unique and overwhelming.

With patterns:

  • “This is a Two Pointers problem"
  • "I need Sliding Window here"
  • "This calls for Modified Binary Search”

Result: Solve problems 10x faster by recognizing familiar patterns.

Pattern 1: Two Pointers

When to use:

  • Array/string is sorted or you need to find pairs
  • Problems involving pairs, triplets, or subarrays
  • Compare elements from both ends

How it works:

  • Two pointers moving toward each other or in the same direction
  • Reduces O(n²) to O(n)

Common Problems

Recognize this pattern when you see:

  • “Find pair/triplet that sums to target"
  • "Remove duplicates from sorted array"
  • "Reverse array/string"
  • "Palindrome checking"
  • "Container with most water”

Pattern 2: Sliding Window

When to use:

  • Finding subarrays/substrings with specific properties
  • Problems asking for “maximum/minimum subarray"
  • "Contiguous” elements

How it works:

  • Maintain a window [left, right]
  • Expand window by moving right
  • Shrink window by moving left when condition breaks

Common Problems

Recognize this pattern when you see:

  • “Longest/shortest substring with…"
  • "Maximum/minimum subarray sum"
  • "Find all anagrams in string"
  • "Smallest window containing…”
  • Keywords: contiguous, subarray, substring

Pattern 3: Fast & Slow Pointers (Tortoise & Hare)

When to use:

  • Linked list cycle detection
  • Finding middle of linked list
  • Detecting patterns in sequences

How it works:

  • Two pointers moving at different speeds
  • Fast pointer moves 2 steps, slow pointer moves 1 step

Common Problems

Recognize this pattern when you see:

  • “Detect cycle in linked list"
  • "Find middle of linked list"
  • "Find start of cycle"
  • "Happy number problem"
  • "Palindrome linked list”

Pattern 4: Merge Intervals

When to use:

  • Overlapping intervals
  • Scheduling problems
  • Range merging

How it works:

  • Sort intervals by start time
  • Merge overlapping intervals

Common Problems

Recognize this pattern when you see:

  • “Merge overlapping intervals"
  • "Insert interval"
  • "Meeting rooms"
  • "Employee free time”
  • Keywords: intervals, ranges, scheduling

Pattern 5: Cyclic Sort

When to use:

  • Array contains numbers in given range (1 to n)
  • Finding missing/duplicate numbers

How it works:

  • Place each number at its correct index
  • One pass to sort, one pass to find anomalies

Common Problems

Recognize this pattern when you see:

  • “Numbers in range 1 to n"
  • "Find missing/duplicate number"
  • "Find all duplicates"
  • "Find corrupt pair”

Pattern 6: In-place Reversal of Linked List

When to use:

  • Reversing linked list (whole or partial)
  • Modifying linked list structure

How it works:

  • Reverse pointers one by one
  • Track previous, current, next nodes

Common Problems

Recognize this pattern when you see:

  • “Reverse linked list"
  • "Reverse sublist"
  • "Reverse nodes in k-group"
  • "Rotate list”

Pattern 7: Tree BFS (Breadth-First Search)

When to use:

  • Level-order traversal
  • Finding shortest path in tree
  • Level-by-level processing

How it works:

  • Use a queue
  • Process nodes level by level

Common Problems

Recognize this pattern when you see:

  • “Level order traversal"
  • "Zigzag traversal"
  • "Minimum depth of tree"
  • "Right/left side view"
  • "Level averages”

Pattern 8: Tree DFS (Depth-First Search)

When to use:

  • Path-related problems
  • Tree traversal (preorder, inorder, postorder)
  • Searching in tree depth

How it works:

  • Use recursion or stack
  • Explore one branch fully before backtracking

Common Problems

Recognize this pattern when you see:

  • “Find all paths"
  • "Path with given sum"
  • "Diameter of tree"
  • "Maximum depth"
  • "Serialize/deserialize tree”

Pattern 9: Two Heaps

When to use:

  • Finding median of stream
  • Dividing elements into two parts

How it works:

  • Max heap for smaller half
  • Min heap for larger half
  • Balance heaps to maintain median

Common Problems

Recognize this pattern when you see:

  • “Find median of stream"
  • "Sliding window median”
  • Keywords: median, middle element, stream

Pattern 10: Subsets / Backtracking

When to use:

  • Generate all combinations/permutations
  • Explore all possibilities
  • Problems with multiple choices at each step

How it works:

  • Make a choice, recurse, undo choice (backtrack)
  • Build solution incrementally

Common Problems

Recognize this pattern when you see:

  • “Generate all subsets/combinations"
  • "Generate permutations"
  • "N-Queens problem"
  • "Sudoku solver"
  • "Word search”
  • Keywords: all possible, combinations, permutations

Pattern 11: Modified Binary Search

When to use:

  • Sorted or rotated sorted array
  • Finding element in O(log n)
  • “First/last occurrence” problems

How it works:

  • Binary search with modifications
  • Handle edge cases (rotation, duplicates)

Common Problems

Recognize this pattern when you see:

  • “Search in rotated sorted array"
  • "Find first/last occurrence"
  • "Search in infinite array"
  • "Find peak element”
  • Keywords: sorted, O(log n), search

Pattern 12: Top K Elements

When to use:

  • Finding K largest/smallest elements
  • Frequency-based problems

How it works:

  • Use heap of size K
  • Keep track of top K elements

Common Problems

Recognize this pattern when you see:

  • “K largest/smallest elements"
  • "Top K frequent elements"
  • "K closest points"
  • "Kth largest element”
  • Keywords: top K, K largest, K most frequent

Pattern 13: K-way Merge

When to use:

  • Merge K sorted arrays/lists
  • Problems involving multiple sorted inputs

How it works:

  • Use min heap to track smallest element from each array
  • Extract min, add next element from that array

Common Problems

Recognize this pattern when you see:

  • “Merge K sorted lists"
  • "Kth smallest in sorted matrix"
  • "Smallest range covering K lists”
  • Keywords: K sorted, merge K

Pattern 14: Dynamic Programming Patterns

When to use:

  • Overlapping subproblems
  • Optimal substructure
  • ”Maximum/minimum” with constraints

Common DP Patterns

A. 0/1 Knapsack

B. Fibonacci-style (Linear DP)

C. Longest Common Subsequence

Recognize DP when you see:

  • “Maximum/minimum cost"
  • "Count number of ways"
  • "Longest/shortest subsequence"
  • "Can you reach…”
  • Keywords: optimal, maximum, minimum, count ways

  │
  ├─ Array/String problems?

  │   ├─ Subarray/Substring? → Sliding Window
  │   ├─ Numbers 1 to n? → Cyclic Sort
  │   └─ Find K elements? → Top K / Heap
  │
  ├─ Linked List problems?
  │   ├─ Cycle detection? → Fast & Slow Pointers
  │   └─ Reversal? → In-place Reversal
  │
  ├─ Tree problems?
  │   ├─ Level-order? → Tree BFS
  │   ├─ Path problems? → Tree DFS
  │   └─ Median/balance? → Two Heaps
  │
  ├─ Intervals?
  │   └─ Overlapping? → Merge Intervals
  │
  ├─ Generate all?
  │   └─ Combinations/Permutations → Backtracking
  │
  ├─ Sorted + O(log n)?
  │   └─ Modified Binary Search
  │
  ├─ K sorted inputs?
  │   └─ K-way Merge
  │
  └─ Optimization problem?
      └─ Overlapping subproblems? → Dynamic Programming

The Recognition Process

Step 1: Read the Problem

Identify keywords: “subarray”, “sorted”, “K largest”, “all combinations”, etc.

Step 2: Ask Questions

  • Is input sorted?
  • Do I need all solutions or just one?
  • What’s the constraint? (time/space)
  • Is there a range (1 to n)?

Step 3: Match Pattern

Use the flowchart or table above.

Step 4: Adapt Template

Start with pattern template, adjust for specific requirements.

Step 5: Code & Test

Implement and verify with examples.

Key Takeaway

Master patterns, not problems.

  • Each pattern is a tool in your toolbox
  • Problems are just variations of these patterns
  • Recognition comes with practice

The goal: See a new problem → Recognize pattern → Apply template → Solve in minutes
