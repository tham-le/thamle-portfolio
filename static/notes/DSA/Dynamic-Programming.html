Dynamic Programming Mastery Guide

Dynamic Programming (DP) transforms exponential problems into polynomial ones. It’s the difference between unusable and production-ready.

This guide covers DP patterns, techniques, and classic problems to master this powerful approach.

What Is Dynamic Programming?

Definition

Breaking complex problems into overlapping subproblems, solving each once, and storing solutions (memoization/tabulation).

Two Key Properties

1. Optimal Substructure

Optimal solution contains optimal solutions to subproblems.

Example: Shortest path
Path(A → D) = Path(A → B) + Path(B → D)
If A → D is optimal, then A → B must also be optimal.

2. Overlapping Subproblems

Same subproblems solved multiple times.

Fibonacci:
fib(5) = fib(4) + fib(3)
fib(4) = fib(3) + fib(2)
         └─────┘
         Overlap! fib(3) calculated twice

Memoization vs Tabulation

AspectMemoizationTabulation
ApproachTop-down (recursive)Bottom-up (iterative)
ImplementationRecursion + cacheLoops + table
SpaceRecursion stack + cacheJust table
When to solveOnly needed subproblemsAll subproblems
Easier forComplex recursionSpace optimization

Rule of thumb: Start with memoization (easier), optimize to tabulation if needed.

DP Pattern 1: Fibonacci-Style (Linear DP)

Pattern

Each state depends on previous states in sequence.

Formula

dp[i] = f(dp[i-1], dp[i-2], ...)

Classic Problems

1. Fibonacci Numbers

2. Climbing Stairs

3. House Robber

4. Min Cost Climbing Stairs

DP Pattern 2: 0/1 Knapsack

Pattern

For each item, make binary choice: include or exclude.

Formula

dp[i][w] = max(exclude, include)

  • exclude = dp[i-1][w]
  • include = value[i] + dp[i-1][w - weight[i]]

Classic Problems

1. 0/1 Knapsack

2. Partition Equal Subset Sum

3. Target Sum

4. Coin Change

DP Pattern 3: Longest Common Subsequence (LCS)

Pattern

Compare two sequences, find optimal alignment.

Formula

if s1[i] == s2[j]:
    dp[i][j] = 1 + dp[i-1][j-1]
else:
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])

Classic Problems

1. Longest Common Subsequence

2. Edit Distance (Levenshtein)

3. Longest Palindromic Subsequence

4. Longest Increasing Subsequence (LIS)

DP Pattern 4: Grid/Matrix Paths

Pattern

Find paths in 2D grid, often with obstacles or costs.

Formula

dp[i][j] = f(dp[i-1][j], dp[i][j-1])

Classic Problems

1. Unique Paths

2. Unique Paths II (with obstacles)

3. Minimum Path Sum

4. Dungeon Game

DP Pattern 5: String Partition

Pattern

Break string into parts, optimize some property.

Classic Problems

1. Word Break

2. Palindrome Partitioning II

DP Pattern 6: State Machine

Pattern

States with transitions, optimize across state changes.

Classic Problems

1. Buy and Sell Stock with Cooldown

How to Solve DP Problems

Step-by-Step Framework

1. Identify DP Problem

Look for:

  • “Maximum/minimum"
  • "Count ways"
  • "Longest/shortest"
  • "Can you…”
  • Multiple ways to solve subproblems

2. Define State

What information do we need to track?

  • dp[i]: Answer for first i elements
  • dp[i][j]: Answer for range i to j
  • dp[i][j][k]: Multiple parameters

3. Find Recurrence Relation

How does current state depend on previous states?

  • Look at last decision made
  • Consider all possibilities

4. Determine Base Cases

What are the simplest cases?

  • Empty input
  • Single element
  • Edge boundaries

5. Choose Implementation

  • Start with recursive + memoization
  • Optimize to iterative + tabulation
  • Further optimize space if needed

Key Takeaways

Dynamic Programming is about optimal decisions:

  • Identify: Overlapping subproblems + optimal substructure
  • Define: State representation
  • Formulate: Recurrence relation
  • Implement: Memoization or tabulation
  • Optimize: Space and time

Master DP, and exponential problems become polynomial.
