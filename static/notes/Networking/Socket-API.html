WTF are bind() and accept()?

bind() and accept() are Socket API calls used to create network servers:

  • bind() - Assigns an IP address and port to a Socket
  • accept() - Accepts an incoming connection

Together with socket() and listen(), they form the foundation of server programming.

The Phone Call Analogy

Starting a Business Phone Line

socket() - Get a Phone

”I need a phone to receive calls”

bind() - Get a Phone Number

”Assign this phone number (555-1234) to my phone”

Now people can call you at 555-1234

listen() - Answer Machine Ready

”I’m ready to receive calls, queue up to 10 callers”

accept() - Answer the Phone

”Ring ring!” (incoming call)

Now you can talk to the caller

That’s the server socket lifecycle - bind assigns your address, accept handles connections.

The Socket API Flow

Server Side

1. socket()  → Create socket
2. bind()    → Assign IP:Port
3. listen()  → Mark as passive (server socket)
4. accept()  → Wait for connections (blocks)
   ↓
   Connection arrives
   ↓
5. accept() returns new socket for this client
6. send()/recv() → Communicate
7. close() → End connection
   ↓
   Back to step 4 (accept next connection)

Client Side

1. socket()  → Create socket
2. connect() → Connect to server IP:Port
3. send()/recv() → Communicate
4. close() → End connection

accept() - Accept Incoming Connections

What It Does

Purpose: Wait for incoming connection and return a new socket for communication.

Behavior: Blocks until a connection arrives (unless socket is non-blocking).

The Big Takeaway

bind() and accept() in a Nutshell

bind(sockfd, addr, addrlen)

  • Assigns IP address + port to a Socket
  • Must be called before listen()
  • Address options: 0.0.0.0 (all), 127.0.0.1 (localhost), or specific IP

accept(sockfd, addr, addrlen)

  • Waits for incoming connection (blocks by default)
  • Returns NEW socket for client communication
  • Original socket continues accepting connections

Server flow:

  1. socket() - Create socket
  2. bind() - Assign address
  3. listen() - Mark as passive
  4. accept() - Wait for clients (loop)
  5. send()/recv() - Communicate
  6. close() - End connection

Common patterns:

  • Single-threaded: Accept loop (one client at a time)
  • Multi-threaded: Thread per client
  • Event-driven: epoll + Non-blocking-IO (C10k-Problem solution)

The rule: bind() tells the OS which address/port your server uses. accept() waits for clients and returns a new socket for each connection. For high concurrency, use Non-blocking-IO with epoll instead of blocking accept().
