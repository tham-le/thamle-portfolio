Network-Layer

It’s the post office of the internet

The Postal Service Analogy

You write a letter to your friend across the country. You put it in an envelope with their address. You drop it in a mailbox.

What happens next? Magic.

The postal service doesn’t care what’s in your envelope. They look at the destination address, figure out the route (maybe through Chicago, then Denver, then San Francisco), and pass it along from sorting facility to sorting facility until it reaches your friend’s mailbox.

That’s the Network Layer. It’s Layer 3 in the networking stack, and its only job is: get this packet from here to there, across potentially dozens of networks.

Why Do We Need This Layer?

Imagine you’re in New York and want to talk to a server in Tokyo. Between you and that server are:

  • Your home WiFi network
  • Your ISP’s network
  • A backbone provider’s network
  • An undersea cable network
  • A Japanese ISP’s network
  • The server’s datacenter network

Each of these is a separate network with different technologies, different administrators, and different rules.

The Network Layer solves this: how do we move data across multiple independent networks?

Without it, your computer would need to know the exact path to every destination on the internet. There are billions of devices. Good fucking luck.

The Key Insight: Logical Addresses

The Network Layer introduces the concept of logical addresses that work across different physical networks.

Your Ethernet network uses MAC addresses. But MAC addresses are only meaningful on your local network. Your router’s MAC address means nothing to a router in Tokyo.

So the Network Layer uses IP addresses. These are hierarchical, globally unique addresses that routers can use to make forwarding decisions.

192.168.1.100   → Your computer (private network)
8.8.8.8         → Google's DNS (public internet)
172.217.14.206  → Google's web server (public internet)

Think of it this way:

  • MAC address = your apartment number (only meaningful in your building)
  • IP address = your full mailing address (meaningful anywhere in the world)

What the Network Layer Does

The Network Layer has three main responsibilities:

1. Addressing

Every device needs a unique IP address so packets know where to go.

IPv4 gives us addresses like 192.168.1.1 (32 bits).
IPv6 gives us addresses like 2001:0db8::1 (128 bits, because we ran out of IPv4 addresses).

2. Routing

Routers look at the destination IP address and decide: “Which of my neighbors should I send this to?”

They maintain routing tables that say things like:

  • Packets for 10.0.0.0/8 → send to interface eth0
  • Packets for 192.168.1.0/24 → send to interface eth1
  • Everything else → send to gateway 203.0.113.1

This is how your packet hops from router to router across the internet.

3. Forwarding

Once a router decides where to send a packet, it actually does it. This involves:

  • Decreasing the TTL (time to live) by 1
  • Recalculating the checksum
  • Encapsulating the packet in a new Ethernet frame for the next hop
  • Sending it out the right interface

IP: The Star of the Show

The protocol that runs the Network Layer is IP (Internet Protocol). Currently, we use two versions:

IPv4 (the old guard): 32-bit addresses, been around since 1983
IPv6 (the future): 128-bit addresses, slowly taking over

The transition is slow mainly because IPv4 still works, thanks to technologies like NAT that extend its life.

The IPv4 Packet Structure

Let me show you what an IP packet actually looks like:

The most important fields:

TTL (Time To Live): Prevents packets from circling forever. Every router decreases TTL by 1. When it hits 0, the packet dies. Usually starts at 64.

Protocol: Tells the receiver what’s inside. 6 means TCP, 17 means UDP, 1 means ICMP.

src_ip / dst_ip: Where it’s from, where it’s going. The router only looks at dst_ip to make forwarding decisions.

Checksum: Makes sure the header didn’t get corrupted in transit. If it fails, the packet is dropped.

How Routing Actually Works

Let’s trace a packet from your computer (192.168.1.100) to a web server (93.184.216.34).

Step 1: Your Computer

Your computer looks at the destination IP. It asks: “Is this on my local network?”

Your computer knows:

  • My IP: 192.168.1.100
  • My subnet mask: 255.255.255.0
  • Local network range: 192.168.1.0 to 192.168.1.255

93.184.216.34 is NOT in that range. So your computer says: “Not local. Send it to the default gateway.”

Your default gateway is usually your home router: 192.168.1.1.

Step 2: Your Home Router

Your router receives the packet. It looks at the destination: 93.184.216.34.

The router checks its routing table:

Destination         Gateway          Interface
192.168.1.0/24      0.0.0.0          eth0 (local)
0.0.0.0/0           203.0.113.1      eth1 (ISP)

The routing table says: “For everything else (0.0.0.0/0 means “default route”), send it to my ISP at 203.0.113.1.”

The router:

  1. Decreases TTL by 1
  2. Recalculates the checksum
  3. Wraps the packet in a new Ethernet frame for the next hop
  4. Sends it to the ISP

Step 3: ISP Routers

Your packet hops through multiple ISP routers. Each one:

  1. Strips the Ethernet frame
  2. Looks at the destination IP
  3. Consults its routing table
  4. Forwards to the next router
  5. Wraps in a new Ethernet frame

These routers have bigger routing tables with more specific routes. They might know: “For 93.184.0.0/16, send to router X.”

Step 4: Destination Network

Eventually, the packet reaches the network where 93.184.216.34 lives. The final router sees: “This is on my local network!” and delivers it directly to the server.

The server receives the packet, strips off the IP header, and passes the TCP segment upward.

Routing Tables in Detail

Let’s look at a real routing table. On Linux, run:

You might see:

default via 192.168.1.1 dev wlan0 proto dhcp metric 600
192.168.1.0/24 dev wlan0 proto kernel scope link src 192.168.1.100 metric 600

Breaking this down:

Line 1: The default route. Any destination that doesn’t match another rule goes to 192.168.1.1 (your router) via interface wlan0.

Line 2: Local network route. Traffic for 192.168.1.0/24 stays on the local network. No gateway needed.

Routers on the internet backbone have massive routing tables with hundreds of thousands of routes. They use complex algorithms like BGP (Border Gateway Protocol) to figure out the best paths.

The Big Problem: IPv4 Exhaustion

IPv4 gives us 32-bit addresses. That’s 2^32 = 4.3 billion addresses.

Sounds like a lot, right? Wrong.

We have more than 8 billion people on Earth. Many have multiple devices (phone, laptop, tablet, smart TV). Add in servers, routers, IoT devices… and we’re fucked.

The Internet Assigned Numbers Authority already allocated the last blocks of IPv4 address space to regional Internet registries, and by 2020, Europe’s regional registry had depleted its IPv4 pool.

The Bandaid: NAT

NAT (Network Address Translation) lets multiple devices share one public IP address. Your home router has one public IP, but all your devices have private IPs like 192.168.1.x.

When you make a request, your router rewrites the source IP to its public IP and remembers the mapping. When the response comes back, it rewrites it again and sends it to the right device.

NAT works, but it’s a hack. It breaks the end to end principle of the internet. IPv6 is the real solution.

Subnetting: Dividing Networks

IP addresses are hierarchical. The Network Layer uses subnetting to divide networks into smaller chunks.

An IP address is split into two parts:

  • Network portion: Identifies the network
  • Host portion: Identifies the device on that network

The subnet mask tells you where the split is:

IP:          192.168.1.100
Subnet mask: 255.255.255.0
             ^^^^^^^^^^^^^ network portion
                        ^ host portion

In CIDR notation: 192.168.1.100/24 (the /24 means the first 24 bits are the network portion).

This network can have 2^8 = 256 addresses (.0 to .255). Usually:

  • .0 is the network address (not usable)
  • .255 is the broadcast address (not usable)
  • .1 is often the router
  • .2 to .254 are available for hosts

Subnetting lets organizations divide their IP space efficiently. A company with a /20 network can split it into 16 /24 networks for different departments.

Special IP Addresses

Some IP ranges have special meanings:

Private networks (not routable on the public internet):

  • 10.0.0.0/8 (10.x.x.x)
  • 172.16.0.0/12 (172.16.x.x to 172.31.x.x)
  • 192.168.0.0/16 (192.168.x.x)

Loopback: 127.0.0.0/8 (usually just 127.0.0.1) routes back to yourself

Link local: 169.254.0.0/16 automatically assigned when DHCP fails

Broadcast: 255.255.255.255 sends to everyone on the local network

Multicast: 224.0.0.0/4 sends to a group of hosts

ICMP: The Network Layer’s Messenger

The Network Layer also includes ICMP (Internet Control Message Protocol). It’s used for error messages and diagnostics.

Ping uses ICMP Echo Request/Reply messages.
Traceroute uses ICMP Time Exceeded messages to map the path to a destination.

When a packet can’t be delivered, routers send ICMP messages back:

  • Destination Unreachable: Network is down, host is down, port is closed
  • Time Exceeded: TTL reached 0
  • Redirect: There’s a better route

The Big Picture

The Network Layer is the glue that holds the internet together. It:

  • Provides a global addressing scheme (IP addresses)
  • Enables routing across independent networks
  • Handles fragmentation when packets are too big
  • Manages TTL to prevent infinite loops

Without it, we’d be stuck on our local networks, unable to reach the wider internet.

The Network Layer's Job Take a packet, look at the destination IP, figure out the next hop, and forward it along. Repeat at every router until it reaches its destination. Simple concept, massive scale.
