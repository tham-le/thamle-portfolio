WTF is a Router?

WTF is a Router?

Nov 06, 20256 min read

  • networking
  • infrastructure
  • fundamentals

A router is a network device that forwards Packets between different networks. It’s like a traffic cop at an intersection, directing cars (data) to the right road (network).

Routers operate at Layer 3 (IP layer) and make decisions based on IP addresses.

The Post Office Analogy

The Postal System

Your Home Network (Local Post Office)

Your computer sends a letter (packet) addressed to 8.8.8.8 (Google DNS).

Router (Post Office Sorting Center)

The router looks at the destination address:

  • 8.8.8.8? Not on this local network
  • Check routing table: “Send to ISP”
  • Forwards packet to ISP’s network

ISP Router (Regional Sorting Center)

  • Looks at destination: 8.8.8.8
  • Check routing table: “Send to Internet backbone”
  • Forwards to next router

More Routers (Distribution Centers)

Packet hops through multiple routers until it reaches Google’s network.

Destination Router (Final Post Office)

Delivers packet to 8.8.8.8 (Google’s server).

Each router only knows the “next hop” - not the full path.

What a Router Does

1. Forwarding Packets

Packet arrives: Dest IP = 192.168.2.50

Router checks routing table:
┌──────────────────┬────────────┬──────────┐
│ Destination      │ Gateway    │ Interface│
├──────────────────┼────────────┼──────────┤
│ 192.168.1.0/24   │ Direct     │ eth0     │  ← Local network
│ 192.168.2.0/24   │ Direct     │ eth1     │  ← Match! Send via eth1
│ 0.0.0.0/0        │ 10.0.0.1   │ eth2     │  ← Default route (Internet)
└──────────────────┴────────────┴──────────┘

→ Forwards packet out eth1 interface

Decision: Which interface/gateway to send the packet through?

2. Connecting Different Networks

Home Network (192.168.1.0/24)
    ↓
  Router ← You are here
    ↓
Internet (0.0.0.0/0)

Without a router: Devices on different networks can’t communicate.

3. NAT (Network Address Translation)

Private IP (192.168.1.10) → Router → Public IP (203.0.113.5)
                                     ↓
                                 Internet sees router's public IP

Why? IPv4 address exhaustion - multiple devices share one public IP.

4. DHCP (Assigning IP Addresses)

New device connects
→ Router: "Here's your IP: 192.168.1.105"
→ Device: "Thanks!"

Most home routers act as DHCP servers.

5. Firewall

Incoming packet: Source = 8.8.8.8, Dest Port = 22 (SSH)

Router firewall rules:
- Block all incoming SSH? → ❌ DROP

Routers can filter traffic for security.

Routing Table

Example Routing Table

Interpretation:

  • 0.0.0.0/0: Default route - send everything else to 192.168.1.1 (your home router)
  • 192.168.1.0/24: Direct - devices on local network, no gateway needed
  • 10.0.0.0/8: Send to 192.168.1.254 (VPN gateway)

How Routing Works

Packet destination: 8.8.8.8

Check routing table:
1. 192.168.1.0/24? NO
2. 10.0.0.0/8? NO
3. 0.0.0.0/0 (default)? YES → Send to 192.168.1.1

Longest prefix match: Most specific route wins.

Types of Routers

1. Home Router

Your ISP → Modem → Router → Your devices

Features:

  • NAT (shares one public IP)
  • DHCP (assigns private IPs)
  • Wi-Fi access point
  • Basic firewall
  • Usually all-in-one device

Example: TP-Link, Netgear, ASUS routers

2. Enterprise Router

Company Network → Core Router → ISPs / Branch offices

Features:

  • Multiple WAN connections
  • Advanced routing protocols (BGP, OSPF)
  • VPN support
  • Quality of Service (QoS)
  • High throughput (10-100 Gbps)

Example: Cisco, Juniper routers

3. Core Internet Router

ISP A ↔ Core Router ↔ ISP B

Features:

  • BGP (Border Gateway Protocol)
  • Terabit throughput
  • Redundancy
  • Routes traffic between ISPs

Example: Cisco CRS, Juniper PTX

Router vs. Switch vs. Modem

DeviceLayerFunction
RouterLayer 3 (IP)Forwards between networks
SwitchLayer 2 (Ethernet)Forwards within a network
ModemLayer 1 (Physical)Converts signals (cable/fiber ↔ Ethernet)

Example Network

Internet
   ↓
[Modem] ← Converts ISP signal to Ethernet
   ↓
[Router] ← Routes between Internet and home network
   ↓
[Switch] ← Connects multiple devices locally
   ↓
Devices (computers, phones, etc.)

Router connects DIFFERENT networks. Switch connects devices on SAME network.

How Routers Find Routes

Static Routing (Manual)

Use case: Small networks, specific routes.

Dynamic Routing (Automatic)

Routers communicate with each other to learn routes:

Router A: "I can reach 10.0.0.0/8"
Router B: "OK, I'll send 10.0.0.0/8 traffic to you"

Protocols:

  • RIP (Routing Information Protocol) - Simple, distance-vector
  • OSPF (Open Shortest Path First) - Fast convergence, link-state
  • BGP (Border Gateway Protocol) - Internet backbone routing

Packet Journey Through Routers

Example: Visiting example.com (93.184.216.34)

1. Your computer (192.168.1.10)
   → Checks: Is 93.184.216.34 on local network? NO
   → Sends to default gateway (192.168.1.1 - your home router)

2. Home Router (192.168.1.1)
   → NAT: Replace source IP with public IP
   → Routing table: Send to ISP gateway
   → Forwards to ISP

3. ISP Router
   → Routing table: Send to Internet backbone
   → Forwards to Tier 1 ISP

4. Tier 1 ISP Core Router
   → BGP: Knows example.com's network
   → Forwards toward example.com's network

5. example.com's Router
   → Destination reached!
   → Forwards to server 93.184.216.34

Each hop: Router decrements TTL (Time To Live) field. If TTL=0, packet dies (prevents infinite loops).

The Big Takeaway

Router in a Nutshell

A router forwards Packets between different networks by:

  • Looking at destination IP address
  • Consulting routing table
  • Forwarding to next hop (gateway)

Key functions:

  • Connect different networks
  • NAT (share one public IP among many devices)
  • DHCP (assign IP addresses)
  • Firewall (filter traffic)

How routing works:

  • Routing table: Destination → Gateway → Interface
  • Longest prefix match
  • Default route (0.0.0.0/0) for unknown destinations

Router vs. Switch:

  • Router: Between networks (Layer 3, IP addresses)
  • Switch: Within network (Layer 2, MAC addresses)

The rule: Routers connect different networks and forward Packets based on IP addresses. Every hop on the internet goes through a router. Your home router connects your private network (192.168.x.x) to the public internet.
